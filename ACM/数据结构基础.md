# 一、链表

单链表在算法题中最多的应用是邻接表，存储图和树，以供最短路、最大流、最小生成树等算法使用

单链表由节点组成，每个节点包含值和下一个节点的位置

以下代码实现中，每个节点包含值`val`和指向下一个节点的指针`ne`，链表头`head`仅仅是一个指针，初始化时赋为$-1$，只要指针指向了$-1$，就代表链表结束。在未进行操作时，`idx`永远指向下一个没有赋值的数组

### 数组实现

```C++
const int N=1e6;
int head=-1,val[N],ne[N],idx=0;
// 在链表头和第一个节点之间插入一个数 a
void insertH(int a){
    val[idx]=a, ne[idx]=head, head=idx++;
}
// 将第一个节点删除，需要保证第一个节点存在
void removeH(){
    head=ne[head];
}
// 将 a 插入在下标为 k 的节点的后面
void insertK(int k,int a){
    val[idx]=a, ne[idx]=ne[k], ne[k]=idx++;
}
// 将下标为 k 的节点的后面一个点删除
void removeK(int k){
    ne[k]=ne[ne[k]];
}
// 找到第 n 个节点的下标
int findIt(int n){
    int it=head;
    for(int i=1;i<n;i++)
        it=ne[it];
    return it;
}
// 找到第 n 个节点的值
int findVal(int n){
    return val[findIt(n)];
}
// 将 a 插入在第 n 个节点的后面
void insertN(int n,int a){
    insertK(findIt(n),a);
}
// 将第 n 个节点的后面一个点删除
void removeN(int n){
    removeK(findIt(n));
}
```

- 以上代码皆没有考虑边界和非法操作问题，如果需要考虑的话，要注意`idx`和`it`是否会大于等于 N ，要注意判断指针是否指向了 $-1$

- 在邻接表存图时，不需要开n个数组存n个单链表，只需要定义head数组即可

# 二、双向链表

双向链表由节点组成，每个节点包含值、上一个节点的位置、下一个节点的位置。

### 数组实现

在以下代码实现中，每个节点包含值`val`和指向左右节点的指针`l` 、`r`，`idx`表示当前用到了哪个节点，使用了下标为0和1的节点作为头尾，没有使用链表头

```C++
const int N=1e6;
int val[N], l[N], r[N], idx;
// 初始化，0当作左端点，1当作右端点
void init(){
    r[0]=1, l[1]=0, idx=2;
}
// 在下标为 k 的节点的右边插入一个数 a
void insert(int k, int a){
    val[idx]=a;
    l[idx]=k, r[idx]=r[k];
    l[r[k]]=idx, r[k]=idx++;
    // 注意这里的顺序，先把 idx 节点指好
}
// 删除下标为 k 的节点
void remove(int k){
    l[r[k]] = l[k];
    r[l[k]] = r[k];
}
```

    s.pop();
    s.pop();

    // 检查栈是否为空
    if (s.empty()) {
        std::cout << "Stack is empty." << std::endl; // 输出: Stack is empty.
    }

    return 0;

### 单调栈

问题：若有一个数组，要找到每一个数 x 左侧（从 x 往左寻找）第一个比 x 小的数

在这个数组中，若出现了 $a_x\geq a_y$ 且 $x<y$ 则可以删除 $a_x$ 元素，因为当寻找 x 左侧比 x 更小的数的时候永远不可能选择到 $a_x$

那么也就可以用一个栈代替这个数组，每当放入一个新的数的时候，如果这个数小于等于栈顶，就将其弹出，这样输入所有数之后，整个栈的元素是单调递增的，故称单调栈

需要查找整个数组所有左侧第一个更小的数的时候，暴力双重循环对于每个元素都需要遍历数组，时间复杂度为 $O(n^2)$ ，对于单调栈算法，每个元素最多压入和弹出一次，时间复杂度为 $O(2n)$

```C++

int tt = 0;
for (int i = 1; i <= n; i ++ )
{
    while (tt && check(stk[tt], i)) tt -- ;
    stk[ ++ tt] = i;
}
```

单调栈和单调队列问题一般输入量都很大，记得用 scanf 或关闭 sync_with_stdio

### 单调栈例题

> [!info] [USACO09MAR] Look Up S - 洛谷  
>  
> [https://www.luogu.com.cn/problem/P2947](https://www.luogu.com.cn/problem/P2947)  

```C++
\#include <iostream>
\#include <stack>
using namespace std;
const int N=1e5+5;
int main() {
  int n,a[N],res[N];
  stack<int> s;
  cin>>n;
  for(int i=0;i<n;i++) cin>>a[i];
  for(int i=n-1;i>=0;i--){
    while(!s.empty() && a[s.top()]<=a[i]) s.pop();
    res[i]=s.empty()?0:s.top()+1;
    s.push(i);
  }
  for(int i=0;i<n;i++) cout<<res[i]<<endl;
}
```

**单调栈问题若要找左侧，就正序入栈，反之反序。**

- 找左侧时候，要保证塞入一个元素时，左边的元素都访问过，所以自然是正序入栈。

**单调栈问题若要找小值，就构造单调递增栈，反之构造单调递减栈**

- 找小值的时候，在入栈完毕时，前方的元素应该是一个答案，所以只可能更小，所以栈单调递增，反之，招更大值就要单调递减

本题要求找的是右侧第一个更大的数，所以是一个从右侧开始入栈，维护成单调递减的栈

### 单调队列

滑动窗口问题——对于一个数组，有一个长度为 n 的窗口，从左滑动至右（可能是从空窗口开始也有可能从窗口满开始），求每次窗口中的最小值

![[image 4.png|image 4.png]]

维护一个队列，当作窗口内的元素。每次向队尾入队新的数时，将队尾大于该数的都删除，当队首滑出窗口时出队，所以单调队列需要用双向队列实现。在下列代码中使用数组模拟，q数组存的是下标

朴素双重循环解法的时间复杂度大约为 $O(nk)$ ，是用单调队列时间复杂度大约为 $O(n)$

```C++
int hh = 0, tt = -1;
for (int i = 0; i < n; i ++ )
{
    while (hh <= tt && check_out(q[hh])) hh ++ ;  // 判断队头是否滑出窗口
    while (hh <= tt && check(q[tt], i)) tt -- ;
    q[ ++ tt] = i;
}
```

在 O2 优化关闭时，用数组模拟单调队列速度会比 STL 快一些，一般的算法竞赛都不开启 O2 优化，使用数组模拟可能可以过更多的测试点

### 例题

> [!info] 【模板】KMP - 洛谷  
>  
> [https://www.luogu.com.cn/problem/P3375](https://www.luogu.com.cn/problem/P3375)  

下方代码的注释详细解释了 KMP 的代码实现过程

```C++
\#include <bits/stdc++.h>
using namespace std;
int main() {
  
  // s是主串，p是模式串
  // 下标均从1开始，所以在前面加上一个无效字符
  // next数组的意义是，当模式串失配时，略过多少个字符
  string input,s,p;
  cin>>input; s='#'+input;
  cin>>input; p='#'+input;
  int lenp = p.size()-1;
  int lens = s.size()-1;
  vector<int> next;
  next.resize(lenp+1);
  
  // 计算next数组
  // i是模式串的指针，每次循环计算next[i]的值
  // j可以理解为当前已经匹配的字符数量
  for(int i=2,j=0;i<=lenp;i++){
    // 每当失配时候，指针就跳转到next[j]的位置
    // 下标从1开始，略过next[j]个字符，所以直接匹配了j个字符
    // 不过匹配数量归零后，就不能再跳转了
    while(j && p[i]!=p[j+1]) j=next[j];
    // 匹配成功，则匹配成功的字符数量+1
    if(p[i] == p[j+1]) j++;
    // 记录匹配成功的字符数量
    next[i] = j;
  }

  // KMP算法
  // i是主串的指针，j+1是模式串的指针
  // j也可以理解为当前已经匹配的字符数量
  for(int i=1,j=0;i<=lens;i++){
    // 每当失配时候，j+1就跳转到next[j]+1的位置继续匹配
    // 也相当于略过next[j]个字符
    // 不过匹配数量归零后，就不能再跳转了
    while(j && s[i]!=p[j+1]) j=next[j];
    // 匹配成功，则匹配成功的字符数量+1，模式串指针后移
    if(s[i] == p[j+1]) j++;
    // 如果匹配数量等于模式串长度，说明匹配成功
    if(j == lenp){
      j = next[j];
      // 以下是匹配成功后的操作，例如输出匹配成功的位置
      // i-lenp+1 是匹配成功开始的位置
      // i 是匹配成功结束的位置
      cout << i-lenp+1 << endl;
    }
  }
  // 输出next数组
  for(int i=1;i<=lenp;i++)
    cout << next[i] << ' ';
  cout << endl;
  return 0;
}
```

### 堆的基本操作

堆的基本操作是上浮（向上调整`up()`）和下沉（向下调整`down()`）

下沉时，当改变一个节点的键值使其更大的时候，需要比较它的两个子节点，由于需要保证另一个节点比父节点更大，将其与较小的子节点交换位置，如左图，和 3 交换而不是与 4 交换位置。

重复操作，直到该数达到最后一层或同时小于等于两个子节点

上浮时，当改变一个节点的键值使其更小的时候，仅用和其父节点进行比较，若更小则交换位置

重复操作，直到该数达到最顶层或大于等于其父节点

### 堆的基本操作

堆的基本操作都可以通过操作数组和上浮与下沉组合而成

1. 插入一个数：`heap[++size]=x;up(size);`

1. 求集合中的最小值：`heap[1]`

1. 删除最小值：`heap[1]=heap[size];size—;down(1)`

1. 删除任意元素：`heap[k]=heap[size];size—;down(k);up(k);`

1. 修改任意元素：`heap[k]=x;down(k);up(k);`

- `size`用于记录数组当前节点数量

- 删除元素时，现将根节点等同于尾节点，然后删除尾节点，再将根节点直接进行下沉再进行上浮，此时上浮下沉只会有一个操作生效

- 在建堆时，每次都使用插入数的办法时间复杂度会来到 $O(n)$ ，有一种 $O(n)$ 的建堆方式，先将数据全部输入进数组，然后从 n/2 进行下沉操作直到 1，下列模板中有此操作

- 若要求第 k 个插入的点在堆中的位置，可以开 `ph[]` 和 `hp[]` 两个数组，求第 k 个插入的点时访问 `ph[k]`，交换两个元素时需要对这两个数组进行额外操作

```C++
// h[N]存储堆中的值, h[1]是堆顶，x的左儿子是2x, 右儿子是2x + 1
// ph[k]存储第k个插入的点在堆中的位置
// hp[i]存储堆中下标是i的点是第几个插入的
int h[N], ph[N], hp[N], size;

// 交换两个点，及其映射关系
void heap_swap(int a, int b)
{
    swap(ph[hp[a]],ph[hp[b]]);
    swap(hp[a], hp[b]);
    swap(h[a], h[b]);
}

void down(int u)
{
    int t = u;
    if (u * 2 <= size && h[u * 2] < h[t]) t = u * 2;
    if (u * 2 + 1 <= size && h[u * 2 + 1] < h[t]) t = u * 2 + 1;
    if (u != t)
    {
        heap_swap(u, t);
        down(t);
    }
}

void up(int u)
{
    while (u / 2 && h[u] < h[u / 2])
    {
        heap_swap(u, u / 2);
        u >>= 1;
    }
}

// O(n)建堆
for (int i = n / 2; i; i -- ) down(i);
```

# 九、哈希表

> [!info] 哈希表 - OI Wiki  
> OI Wiki 是一个编程竞赛知识整合站点，提供有趣又实用的编程竞赛知识以及其他有帮助的内容，帮助广大编程竞赛爱好者更快更深入地学习编程竞赛  
> [https://oi-wiki.org/ds/hash/](https://oi-wiki.org/ds/hash/)  

哈希函数就是将值运算成键值的函数，这个值可以是整数、浮点数、字符串等等，哈希表就是使用“键值”的对应方式存放数据的数据结构。例如存放 int 范围的整数，哈希函数是 $y=x\mod 10^5$，就能用 y 访问查询是否有 x，但是这样计算会有多个数映射到同样的下标，所以哈希表的关键在于解决冲突（也称哈希碰撞）

- 哈希表主要解决一个问题，快速查找某个值在不在集合内

- 离散化和哈希表的区别：离散化映射在一个连续整数域

### 解决冲突：拉链法

![[QQ_1730423004378.png]]

拉链法也称开散列法（open hashing）

拉链法是在每个存放数据的地方开一个链表，如果有多个键值索引到同一个地方，只用把他们都放到那个位置的链表里就行了

```C++
int h[N], e[N], ne[N], idx;
// 向哈希表中插入一个数
void insert(int x)
{
    int k = (x % N + N) % N;
    e[idx] = x;
    ne[idx] = h[k];
    h[k] = idx ++ ;
}
// 在哈希表中查询某个数是否存在
bool find(int x)
{
    int k = (x % N + N) % N;
    for (int i = h[k]; i != -1; i = ne[i])
        if (e[i] == x)
            return true;

    return false;
}
```

N是哈希函数 $y=x\mod N$ 取模的数。在数学或Python中负数取模运算都是正数，而C++对负数取模还是负数，所以为了实现标准的数学取模使用以下操作取模：`k=(x%N+N)%N`。数组`n[N], ne[N]`用于存储链表，详见前文

### 解决冲突：开放寻址法

若输入 m 个数，则开一个大小为 2m～3m 的数组，插入时，在哈希完毕后直接尝试放入 hash[y]，若该位置已经有数，则寻找下一个位置，直到位置是空的。查找时从哈希后的键值向后找到最后一个非空的位置。

```C++
int h[2*N];

// 如果x在哈希表中，返回x的下标；如果x不在哈希表中，返回x应该插入的位置
int find(int x)
{
    int t = (x % N + N) % N;
    while (h[t] != null && h[t] != x)
    {
        t ++ ;
        if (t == N) t = 0;
    }
    return t;
}
```

### 字符串哈希函数

字符串哈希函数就是将字符串看成一个P进制的数：若字符串仅有26个大写字母组成，假设有字符串”ABCD”，那么就可以将其看成一个27进制的数，ABCD…分别对应了1234…，此时“ABCD”的哈希就是

$$hash(“ABCD”)=(1\times27^3+2\times27^2+3\times27^1+4\times27^0)\mod Q$$

注意不可以把字符映射成 0，在前面的条件下，如果将 ‘A’ 映射成 0，那么”AAAAAB“和”B“的哈希结果将是相同的，这大大提高了冲突概率。在运算时每次算加法（甚至算幂）都要模Q，最终就会得到 0 到 Q-1 的数，不过这样还是太麻烦了，当 Q 取 $2^{64}$ 时，直接使用 unsign long long 类型存储哈希值，那么在运算过程中溢出会自动模 Q

- 一般来说，字符串哈希函数会取 P=131 或 P=13331，Q=$2^{64}$，此时最不容易发生哈希冲突

### 字符串前缀哈希法

字符串前缀哈希法就是将字符串的每一个前缀的哈希值求出，规定`h[0]=0` ，例如字符串“ABCD”:  
`h[0]=0; [1]=hash("A"); h[2]=hash("AB"); h[3]=("ABC"); h[4]=("ABCD");`

得到了每一个前缀的哈希值，就能通过公式计算出该字符串每一个字串的哈希，若要求第 $l$ 个字符到第 $r$个字符的值，则：

$$hash(l\to r)=h[r]-h[l-1]\times P^{r-l+1}$$

在这里的 $h[l-1]\times P^{r-l+1}$ 就是将 $l-1$ 的前缀移动到和 $r$ 前缀对其，例如字符串“ABCDEFG”，要求“CDE”的哈希，只需要拿出“AB”的哈希值h[2]，然后乘 P^{r-l+1} 变成 “AB000” 字符串的哈希值，然后用“ABCDE”的哈希减去“AB000”的哈希，得到“CDE”的哈希。

在运算所有前缀的哈希时，不需要每次取出整个前缀然后对每一位进行运算，只需要将前一位的前缀哈希乘 P 加上该位字符对应的数即可：

$$h[i]=h[i-1]\times P+str[i]$$

字符串前缀哈希的应用：用于快速判断一个超长字符串的两个字串是否相等，尽管会带来 $1/Q$ 的误差，但是 Q 取到 $2^{64}$ 时可以忽略不计，若有碰撞容忍度极低的情况，可以考虑使用双重哈希，也就是用两个不同的哈希函数计算结构同时判断是否相等
