# 一、01背包

有N个物品，物品有重量 $w_i$ 和价值 $v_i$ ,背包总容量 V，每种物品只能选或者不选，求背包内能装入的最大价值

Dp 问题有两个核心，如何表示状态，如何计算状态，每一类 Dp 问题都主要讨论这两个问题。其中状态表示一要考虑如何划分集合，也就是 $i$ 和 $j$ 代表了什么意义，二要考虑属性，也就是 $f(i,j)$ 到底存着什么东西

### 状态表示

使用状态数组 $f(i, j)$ 表示选前 $i$ 个物品，背包最大体积为 $j$ 的情况下储存的属性，属性有三种，一是最大价值，二是最小价值，三是物品数量最多。以下讨论的是最大价值的情况：

### 状态计算

欲计算 $f(i, j)$，可以将其分为两部分，一是不含 $i$ 的情况，此时相当于 $f(i-1, j)$，在数组表中相当于继承上方位置。关键在于另一部分——从 $1$ 到 $i$ 号物品中选择，并且包含 $i$ 的背包最大体积为 $j$ 的情况下的最大价值。由于第 $i$ 号物品是必选的，此时，情况可以变为，从 $1$ 到 $i-1$ 时，背包最大体积为 $j-w_i$ 的情况下的最大价值，此时相当于 $f(i-1,j-w_i)+v_i$。综上所述，可得状态转移方程：

$$f(i,j)=\max\Big(f(i-1,j),f(i-1,j-w_i)+v_i\Big)$$

### 例题

> [洛谷 - [NOIP2005 普及组] 采药](https://www.luogu.com.cn/problem/P1048)  
>  

```C++
\#include <iostream>
using namespace std;
int T,M,f[105][1005];
struct herb{
    int t,v; // time and value
}h[105]; // herbs
int main(){
    cin>>T>>M;
    for(int i=1;i<=M;i++)
        cin>>h[i].t>>h[i].v;
    for(int j=0;j<=T;j++)
        f[0][j]=0;
    for(int i=1;i<=M;i++){
        for(int j=0;j<=T;j++){
            if(j>=h[i].t)
                f[i][j]=max(f[i-1][j],f[i-1][j-h[i].t]+h[i].v);
            else
                f[i][j]=f[i-1][j];
        }
    }
    cout<<f[M][T]<<endl;
}
```

值得注意的是，存储草药的时间和价值时需要用$h[1]$开始，才能匹配状态转移方程中的 $w_i$，此外，需要对 $j-w_i$ 是否可取作判断，如果不行只能继承上面的数

由于每次计算时只用到了 $f(i-1,j)$ 和 $f(i,j)$ 两行，可以用滚动数组交替计算，但还有空间需求更小的方法：

因为每次计算新 $f(i,j)$ 时只用了当前行小于$$ $j$ 的数，若从后往前计算，小于 $j$ 的数还没有被更新过，相当于上一层的数，此外由于不需要手动继承上一行的数，所以只用从尾循环到 $w_i$ 保证能够取 $f(i-1,j-w_i)+v_i$ ，综上所述，可以删去 $f(i,j)$ 中的 $i$ 改成仅用一维数组实现，此时的状态转移方程为：

$$f(j)=\max\Big(f(j),f(j-w_i)+v_i\Big)$$

```C++
\#include <iostream>
using namespace std;
int T,M,f[1005];
struct herb{
    int t,v; // time and value
}h[105]; // herbs
int main(){
    cin>>T>>M;
    for(int i=1;i<=M;i++) cin>>h[i].t>>h[i].v;
    for(int j=0;j<=T;j++) f[j]=0;
    for(int i=1;i<=M;i++)
        for(int j=T;j>=h[i].t;j--)
            f[j]=max(f[j],f[j-h[i].t]+h[i].v);
    cout<<f[T]<<endl;
}
```

滚动数组和以上代码使用的优化方法是 Dp 中重要的优化思想

```C++
for(int i=1;i<=M;i++)
        for(int j=T;j>=h[i].t;j--)
            f[j]=max(f[j],f[j-h[i].t]+h[i].v);
```

# 二、完全背包

有N个物品，物品有重量 $w_i$ 和价值 $v_i$ ,背包总容量 V，每种物品可以选任意多个，求背包内能装入的最大价值

### 状态表示

和01背包完全一致，集合划分的方法是只考虑前 $i$ 个物品，总体积不大于 $j$ ，属性是能存放的物品价值总和的最大值

### 状态计算

在 01 背包中，我们将新的 $f(i,j)$ 划分为两个区域，一半是选择第 $i$ 个物品（选 $1$ 个），另一半是不选择第 $i$ 个物品（选 $0$ 个），在完全背包中，我们将新的 $f(i,j)$ 划分成若干组，分别是选 $0、1、2、3 …… k-1、k$ 个第 $i$ 个物品，这里可以假定 $k → +∞$

对于选择 0 个 $i$ 物品，相当于 $f(i-1,j)$，对于选择了 k 个物品，相当于 $f(i-1,j-k*w_i)+k*v_i$，其实选择 0 个 $i$ 物品也可以归入选择了 k 个物品的情况，综上，可以得到完全背包的状态转移方程：

$$f(i,j)=\max_{k=0}^{k → +∞} \Big(f(i-1,j-kw_i)+kv_i\Big)$$

注意在朴素算法的代码实现中，$kw[ i ]$ 不应大于 $j$，此条件作为第三重循环的执行条件

使用三重循环计算完全背包的状态转移最坏情况下的时间复杂度是 $O(n^3)$，我们可以简单地将其优化至二维，首先列出 $f(i,j)$ 和 $f(i,j-w_i)$ 的所有项：

$$式一：f(i,j)=max\Big(f(i-1,j),f(i-1,j-w_i)+v_i,f(i-1,j-2w_i)+2v_i+f(i-1,j-3w_i)+3v_i+......+f(i-1,j-kw_i)+kv_i\Big)$$

$$式二：f(i,j-w_i)=max\Big(f(i-1,j-w_i),f(i-1,j-2w_i)+v_i,f(i-1,j-3w_i)+2v_i+......+f(i-1,j-(k+1)w_i)+kv_i\Big)$$

不难发现，式一中第2项到最后一项就等于式二加上 $w_i$ ，在 k 趋向无限的情况下可以忽略式二的最后一项，此时则可将式一改为第一项和后面所有项做比较，也就是让式一的第一项和式二进行比较，这样就将完全背包的状态计算优化至了二维：

$$f(i,j)=\max\Big(f(i-1,j),f(i,j-w_i)+v_i\Big)$$

对比一下 01 背包的方程 $f(i,j)=max(f(i-1,j),f(i-1,j-w_i)+v_i)$，不难发现是极为相似的，所以完全背包问题也能够优化为一维：

$$f(j)=\max\Big(f(j),f(j-w_i)+v_i\Big)$$

虽然优化至一维的完全背包状态转移方程和 01背包的完全一致，但是由于要计算的是当前层的 $f(i,j-w_i)+w_i$ 而不是上一层的 $f(i-1,j-w_i)+v_i$，所以第二层循环应该保持正向循环而非反向，直接从 $w_i$ 循环至背包最大容量 V

经过两次的优化，二维算法相比朴素算法的时间复杂度降低了许多，一维算法相比二维算法空间复杂度降低了许多。

> 思考题：为什么01背包和完全背包的一维算法实现仅仅是差了第二层循环的方向？

在代码实现时，仅需要选到`k=V/w[i]+1`即可

# 三、多重背包

有N个物品，物品有重量 $w_i$ 和价值 $v_i$ ,背包总容量 V，每种物品最多选 m 个，求背包内能装入的最大价值

### 状态表示

和01背包完全一致，集合划分的方法是只考虑前 $i$ 个物品，总体积不大于 $j$ ，属性是能存放的物品价值总和的最大值

### 状态计算

多重背包问题和完全背包问题基本一致，只是 k 变为了有限的 m 个，朴素做法的状态转移方程——

$$f(i,j)=\max_{k=0}^{k → m} \Big(f(i-1,j-kw_i)+kv_i\Big)$$

朴素做法的时间复杂度为 $O(NVm)$。多重背包无法使用完全背包的优化方式，若使用同样的优化方式，可以进行如下尝试

$$式一：f(i,j)=max\Big(f(i-1,j),f(i-1,j-w_i)+v_i,f(i-1,j-2w_i)+2v_i+f(i-1,j-3w_i)+3v_i+......+f(i-1,j-mw_i)+mv_i\Big)$$

$$式二：f(i,j-w_i)=max\Big(f(i-1,j-w_i),f(i-1,j-2w_i)+v_i,f(i-1,j-3w_i)+2v_i+......+f(i-1,j-(m+1)w_i)+mv_i\Big)$$

由于 k 不再是无限，而是有限个 m，式一的 2 至 m 项无法用多了一项的式二代替

### 朴素做法例题

> [洛谷 - 樱花](https://www.luogu.com.cn/problem/P1833)  
>  

```C++
\#include <bits/stdc++.h>
using namespace std;
const int N=10005;
const int T=1005;
int main() {
  // 数据输入
  int h1,h2,m1,m2;
  scanf("%d:%d %d:%d",&h1,&m1,&h2,&m2);
  int t=(h2-h1)*60+(m2-m1);
  int n;
  cin>>n;
  int w[N],v[N],m[N];
  for(int i=1;i<=n;i++){
    cin>>w[i]>>v[i]>>m[i];
    // 0表示无限个，处理为有限个
    if(m[i]==0) m[i]=t/w[i]+1;
  }
  // 多重背包朴素解法
  int dp[N][T];
  memset(dp,0,sizeof(dp));
  for(int i=1;i<=n;i++){
    for(int j=0;j<=t;j++){
      for(int k=0;k<=m[i];k++){
        if(j>=w[i]*k) //判断是否越界
          dp[i][j]=max(dp[i][j],dp[i-1][j-w[i]*k]+v[i]*k);
      }
    }
  }
  cout<<dp[n][t]<<endl;
}
```

### 二进制分组优化

多重背包有 m 个相同的物品，可以对其进行二进制分组优化——

若有 1023 个物品，将其打包成 1、2、4、8、16、32、64、128、256、512 一共十包，每一包物品可以选或者不选，那么总是能凑出从 1-1023 任意数量个的物品，这个结论不难证明，其实就是模拟了二进制数，十位二进制数就能表示 0～1023 总共 1024个数

若有 200 个物品，可以将其打包成 1、2、4、8、16、32、64、73 一共八包。推广到 m 个物品，可以将其打包成 $2^0、2^1、2^2、2^3......2^t、c$，其中 $c$ 严格小于等于 $2^{t+1}$

打包完成后，问题就转换为了 0-1背包问题，时间复杂复杂度降低至 $O(NVlogm)$

  

# 四、分组背包

有N组物品，第 $i$ 组物品有 $n$ 个物品，每个物品有重量 $w_i^k$ 和价值 $v_i^k$ ，背包总容量 V，每组物品最多选一个，求背包内能装入的最大价值

### 状态表示

和01背包完全一致，集合划分的方法是只考虑前 $i$ 个物品，总体积不大于 $j$ ，属性是能存放的物品价值总和的最大值

### 状态计算

和多重背包问题类似，计算$f(i,j)$时，问题从“选多少个第 $i$ 个物品“，变成了”第 $i$ 组物品选哪个“，此外再加上不选此物品的情况，也就是 $f(i-1,j)$ ，就能得到以下状态转移方程——

$$f(i,j)=\max\left(f(i-1,j),\max_{k=1}^{k→n} \Big(f(i-1,j-w_i^k)+v_i^k\Big)\right)$$

# 五、线性DP

动态规划的状态表示优先从一维开始考虑，如果无法表示状态，再使用二维，再使用三维

### 例题：数字三角形

> [洛谷 - [USACO1.5] [IOI1994]数字三角形 Number Triangles](https://www.luogu.com.cn/problem/P1216)  
>  

一个数字三角形，用数组 `a[i][j]` 表示，求从上至下经过所有节点的和的最大值

状态表示：一个二维数组，和原数据大小形状相同，$f(i,j)$ 表示由起点走至点 $(i,j)$ 能够得到的最大权值。答案就是数组最后一行的最大值

状态计算：共 n 行，从第 1 行计算至第 n 行，第 i 行，从第 1 个元素计算至第 i 个元素，每个元素只可能从左上元素或者正上方元素走来，故状态转移方程如下：

$$f(i,j)=\max\Big(f(i-1,j-1),f(i-1,j)\Big)+a(i,j)$$

```C++
\#include <bits/stdc++.h>
using namespace std;
const int N=1005;
int main() {
  int n;
  cin>>n;
  int a[N][N];
  int dp[N][N];
  for(int i=1;i<=n;i++)
    for(int j=1;j<=i;j++)
      cin>>a[i][j];
  for(int i=1;i<=n;i++){
    for(int j=1;j<=i;j++){
      if(j==1) dp[i][j]=dp[i-1][j]+a[i][j];
      else if(j==1) dp[i][j]=dp[i-1][j-1]+a[i][j];
      else dp[i][j]=max(dp[i-1][j],dp[i-1][j-1])+a[i][j];
    }
  }
  int res=INT_MIN;
  for(int i=1;i<=n;i++)
    res=max(res,dp[n][i]);
  cout<<res<<endl;
}
```

动态规划的时间复杂度计算就是状态数量乘以状态转移计算量，在上题中，状态转移 $O(n)$

数字三角形也可以从下往上进行状态转移计算，由于只有一个终点，就是顶点，完成后无需遍历

### 例题：最长上升子序列

> [洛谷 - B3637 最长上升子序列](https://www.luogu.com.cn/problem/B3637)  
>  

> 字符串 $S$ 的 **子序列** 是从 $S$ 中将若干元素提取出来并不改变相对位置形成的序列

求最长上升子序列就是从前向后挑出元素形成序列并保证元素递增

状态表示：集合划分——数组 $f_i$ 表示以第 $i$ 个数结尾的上升子序列的集合，属性值——以第 $i$ 个数结尾的上升子序列的数量的最大值。答案就是从 $f_1$ 到 $f_n$ 中取最大值

状态计算：集合 $f_i$ 中，可以分为 $i$ 种情况：

- 第 $i$ 个元素就是序列的开头

- 上一个元素是第 $i-1$ 个数

- 上一个元素是第 $i-2$ 个数

- $\dots$

- 上一个元素是第 $1$ 个数

但是记上一个元素是 $a_j$，当前元素为 $a_i$，如果 $a_j>a_i$，那么这种情况就不存在，反之如果情况存在，那么最长的序列长度就是以 $a_j$ 结尾的最长序列加 $1$，即状态转移方程：

$$f(i)=\max_{j=1}^{i-1}(f(j)+1)$$

```C++
\#include <bits/stdc++.h>
using namespace std;
const int N = 5005;
int main() {
	int n,res=0;
	int a[N],dp[N];
	cin>>n;
	for(int i=0;i<n;i++)
		cin>>a[i];
	for(int i=0;i<N;i++)
		dp[i]=1;
	for(int i=1;i<n;i++)
		for(int j=0;j<i;j++)
			if(a[j]<a[i])
				dp[i]=max(dp[i],dp[j]+1);
	for(int i=0;i<n;i++)
		res=max(res,dp[i]);
	cout<<res<<endl;
}
```

- 初始化 `dp[]` 可以用 `fill()` 函数，要初始化为 1

- 如果要记录序列，就开新数组记录每次更新时的来源位置

该朴素算法的时间复杂度为 $O(n^2)$

### 例题：最长公共子序列

给定两个字符串 $A$ 和 $B$，求一个最长的序列使得该序列同时为两个字符串的子序列

- 经验：涉及到两个子序列、两个字符串的动态规划问题一般用二维状态表示

**状态如何表示？**

集合划分—— $f(i,j)$ 表示在第一个序列的前 $i$ 个字母中出现并且在第二个序列的前 $j$ 个字母中出现的公共子序列，属性——集合内子序列的最长长度

**状态如何计算？**

记字符串 $A$ 的元素为 $a_1,a_2,a_3,\dots$，记字符串 $B$ 的元素为 $b_1,b_2,b_3,\dots$

对于集合 $f(i,j)$ 可以划分成四种情况：

1. 一定选择 $a_i$，一定选择 $b_j$

1. 一定选择 $a_i$，一定不选 $b_j$

1. 一定不选 $a_i$，一定选择 $b_j$

1. 一定不选 $a_i$，一定不选 $b_j$

第一种情况下，一定满足 $a_i=b_j$ 才可以选，选择后序列长度加一，状态即 $f(i-1,j-1)+1$

第四种情况下，直接等于 $f(i-1,j-1)$

第二第三种情况比较难考虑，$f(i-1,j)$ 只能表示选择 $a_{i-1},b_j$ 之前的序列，至于是不是第二种情况中要求的一定包含 $b_j$，不确定

但是 $f(i-1,j)$ 一定包含第二种情况，多余的那些情况其实也一定在第一第三种情况中去出现，所以，虽然第二第三种情况无法直接划分出来，但是由于状态表示的属性是最大值，所以情况划分中允许有重复（但是不可以有漏），而第二第三种情况一定属于在 $f(i-1,j),f(i,j-1)$ 两个集合中

换句话说 $f(i-1,j),f(i,j-1)$，这两个集合涵盖了我们要求最大值的集合的一部分，再加上第一种和第四种情况就涵盖了所有情况，求最大值只需要涵盖到所有情况即可（但是求值不行）

此外一定 $f(i-1,j-1)\in f(i-1,j)\cup f(i,j-1)$，所以第一种情况可以在代码实现中省略，不需要单独求

$$f(i,j)=\max(f(i-1,j),f(i,j-1),f(i-1,j-1)+1)$$

> [洛谷 - 提交记录 191472319](https://www.luogu.com.cn/record/191472319)  
>  

```C++
\#include <bits/stdc++.h>
using namespace std;
const int N = 1e3+10;
int n,a[N],b[N],f[N][N];
int main() {
  cin>>n;
  for(int i=1;i<=n;i++) cin>>a[i];
  for(int i=1;i<=n;i++) cin>>b[i];
  for(int i=1;i<=n;i++){
    for(int j=1;j<=n;j++){
      f[i][j]=max(f[i-1][j],f[i][j-1]);
      if(a[i]==b[j]) f[i][j]=max(f[i][j],f[i-1][j-1]+1);
    }
  }
  cout<<f[n][n]<<endl;
}
```

下面的是将上面的代码改为了滚动数组，因为每次状态计算只需要用前一行的数据，所以可以将所以行号 i 换成 i%2，将所有行号 i-1 换成 !(i%2)

```C++
\#include <bits/stdc++.h>
using namespace std;
const int N = 1e5+10;
int n,a[N],b[N],f[2][N];
int main() {
  cin>>n;
  for(int i=1;i<=n;i++) cin>>a[i];
  for(int i=1;i<=n;i++) cin>>b[i];
  for(int i=1;i<=n;i++){
    for(int j=1;j<=n;j++){
      f[i%2][j]=max(f[!(i%2)][j],f[i%2][j-1]);
      if(a[i]==b[j]) f[i%2][j]=max(f[i%2][j],f[!(i%2)][j-1]+1);
    }
  }
  cout<<f[n%2][n]<<endl;
}
```

不过由于这样的朴素算法时间复杂度仍是 $O(n^2)$，只能在模板题中拿到 60 分

# 六、区间DP

状态表示时表示一段区间就是区间 DP

### 例题：石子合并

> [洛谷 - 石子合并（弱化版）](https://www.luogu.com.cn/problem/P1775)  
>  

有 N 堆石子以及重量 $a_1$ 到 $a_N$，每次操作只能合并相邻两堆石子，代价为两堆石子重量之和，求要将这 N 堆石子合并为一堆的最小代价

**如何表示状态？**

$f(i,j)$ 表示的集合是从第 $i$ 到 $j$ 堆石子合并成一堆的方式，属性是所有合并方式的最小值

**如何计算状态？**

要划分集合，可以考虑合成一堆的上一步，也就是区间 $[i,j]$ 中还有两堆石子的时候，此时一定有一个分界线 $k$ 使得 $a_k$ 在左边区间，$a_{k+1}$ 在右边区间 $k \in [i,j-1]$，也可以理解为上一步合并前左边堆由 $k-i+1$ 堆石子合并得来

由于状态计算时要经常求石子区间的总重量，所以对 $a$ 求前缀和 $pre$，此时假如以 $k$ 为分界线就是最优解，那么相当于两段区间的状态属性再加上整一段 $[i,j]$ 石子重量的和：

$$f(i,j)=f(i,k)+f(k+1,j)+\left[pre(j)-pre(i-1)\right]$$

于是只要将 $k$ 从 $i$ 遍历到 $j-1$ 就能得到状态转移方程：

$$f(i,j)=\min_{k=i}^{j-1}\Big[ f(i,k)+f(k+1,j)+pre(j)-pre(i-1) \Big ]$$

时间复杂度为 $O(n^3)$，在弱化版中 $n\le 300$，最大时间复杂度为 $2.7\times 10^7$

**如何实现代码？**

区间 DP 需要枚举区间长度，代码会比前面讲的 DP 更难实现。由于只有求出区间更小的状态才能求区间更大的状态，所以最外层循环是枚举区间大小，然后枚举起点 $i$，终点 $j$ 自然可以计算到，最后枚举 $k$

```C++
\#include <bits/stdc++.h>
using namespace std;
const int N = 305;
int a[N],dp[N][N],pre[N];
int main() {
  int n;
  cin>>n;
  for(int i=1;i<=n;i++){
    cin>>a[i];
    pre[i]=pre[i-1]+a[i];
  }
  for(int len=1;len<=n;len++){
    for(int i=1;i+len<=n;i++){
      int j=i+len;
      dp[i][j]=0x3f3f3f3f;
      for(int k=i;k<j;k++){
        dp[i][j]=min(dp[i][j],dp[i][k]+dp[k+1][j]+pre[j]-pre[i-1]);
      }
    }
  }
  cout<<dp[1][n]<<endl;
}
```

- len 要从 2 开始，因为一堆石子不需要合并，所以代价就为 0

- $f(i,j)$ 在计算前要初始化，但是不可以在一开始就初始化

# 八、数位统计DP

数位DP 的一种，着重于分情况讨论

### 例题：数字计数

> [洛谷 - [ZJOI2010] 数字计数](https://www.luogu.com.cn/problem/P2602)  
>  

给定 $a,b$ 求 $[a,b]$ 中所有整数数码 $0$ 到 $9$ 出现的次数

基本思路是：实现一个函数 `count(n,x)` 求的是 1 到 n 中数码 $x$ 出现的次数，如果要求 $[a,b]$ 中 $x$ 出现的次数，则求 `count(b,x)-count(a-1,x)`

**怎么求 count 函数？**

举例：假设 $n$ 的每一位是 $ABCDEFG$，假如要求第四位上 $x$ 出现的次数，也就是求 zzz$x$yyy 出现的次数，可以分两种情况：

- 前三位在 $000$ 到 $ABC$ 之间
    
    - 如果 $x$ 不为 $0$，那么后三位就可以在 $000$ 到 $999$ 随便取，一共 $ABC\times 1000$ 种情况
    
    - 如果 $x$ 为 $0$，那么前三位至少要从 $001$ 开始，一共 $(ABC-1)\times 1000$ 种情况
    

- 前三位等于 $ABC$，分三种情况
    
    - $x>D$， 这种情况不在区间内，无法取到
    
    - $x=D$，则后三位可以取 $0$ 到 $EFG$ 一共 $EFG+1$ 种情况
    
    - $x<D$，后三位可以任意取，一共 $1000$ 种情况
    

```C++
\#include <bits/stdc++.h>
using namespace std;
typedef long long ll;
ll count(ll n,int x){
  ll res=0;
  ll k=1;
  while(n>=k){
    ll high=n/(k*10);
    ll low=n%k;
    ll cur=(n/k)%10;
    if(x!=0){
      res+=high*k;
      if(cur==x) res+=low+1;
      if(cur>x) res+=k;
    }else if(high!=0){
      res+=(high-1)*k;
      if(cur==0) res+=low+1;
      if(cur>0) res+=k;
    }
    k*=10;
  }
  return res;
}
int main() {
  ll a,b;
  cin>>a>>b;
  for(int i=0;i<=9;i++){
    cout<<count(b,i)-count(a-1,i)<<" ";
  }
  return 0;
}
```

在上述代码中，k 的作用是取位数，如果 k=100 n=123456789，那么 high=123456，low=89，cur=7
