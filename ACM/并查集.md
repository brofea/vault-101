# 启发式合并

## 按秩合并

常用的办法，每次都选择高度较少的一棵树连接到另外一棵树，避免发生退化

```C++
int p[N]; // p[i] 表示 i 的父节点
int rk[N];  // rk[i] 是以 i 为根的树的近似高度（秩）

// 初始化 n 个元素，每个元素自成一个集合
void dsu_init(int n) {
  for (int i = 1; i <= n; ++i) {
    p[i] = i; // 每个节点的父节点初始是自己
    rk[i] = 0;  // 初始秩为 0
  }
}

// 查找 x 所在集合的代表元素（根节点）
int dsu_find(int x) {
	// 路径压缩：使 x 直接连到根
  if (p[x] != x) p[x] = dsu_find(p[x]); 
  return p[x];
}

// 合并 x 和 y 所在的集合，按秩合并
bool dsu_union(int x, int y) {
  int rx = dsu_find(x), ry = dsu_find(y);
  if (rx == ry)
    return false; // 已在同一集合，无需合并

  // 按秩合并：小树挂到大树下面
  if (rk[rx] < rk[ry]) {
    p[rx] = ry;
  } else if (rk[rx] > rk[ry]) {
    p[ry] = rx;
  } else {
    p[ry] = rx;
    rk[rx]++; // 两棵一样高的树合并，高度+1
  }
  return true; // 成功合并
}
```

按秩合并 + 路径压缩的并查集时间复杂度趋于 $O(\alpha(n))$，其中 $\alpha$ 是阿克曼函数的反函数，阿克曼函数 $A(m,n)$ 是一种增长速度极快的函数，当 $m=4$ 时增长速度就超过指数级增长，所以反函数的时间复杂度极低，在竞赛中 $\alpha(n)$ 几乎不会超过 5，所以上述算法的时间复杂度可看作是线性的

## 按大小合并

每次都选择节点数量较少的一棵树连接到另外一棵树，避免发生退化

在需要集合大小的问题上，适合使用这个，时间复杂度和按秩合并类似

```C++
int parent[MAXN], size[MAXN];

void dsu_init(int n) {
    for (int i = 1; i <= n; ++i) {
        parent[i] = i;
        size[i] = 1; // 每个集合初始大小为1
    }
}

int dsu_find(int x) {
    return parent[x] == x ? x : parent[x] = dsu_find(parent[x]);
}

bool dsu_union(int x, int y) {
    int rx = dsu_find(x), ry = dsu_find(y);
    if (rx == ry) return false;

    // 小集合挂到大集合
    if (size[rx] < size[ry]) {
        parent[rx] = ry;
        size[ry] += size[rx];
    } else {
        parent[ry] = rx;
        size[rx] += size[ry];
    }
    return true;
}
```
