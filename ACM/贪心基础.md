### 例题：区间选择

给定 n 个区间，要求选择最多的区间使得这些区间两两之间没有交集，求选择的数量最大

![[QQ_1736499110275.png]]

1. 将每个区间按右端点从小到大排序

1. 从左到右枚举区间
    
    1. 如果该区间的左端点大于上一次选择区间的右端点，加入
    

**如何证明 cnt 小于等于 ans**

每次选择下一个区间时左端点都在右端点右侧，所以说对于这两个区间来说是不相邻的，此外所有区间按顺序排列，显然，按照上述方法选出的区间一定两两之间都没有交集，这是本体贪心策略的性质，选出的区间也就”是一种可行方案“，而 ans 表示 ”可行方案中数量最大的方案“，所以肯定 cnt 小于等于 ans

**如何证明 cnt 大于等于 ans**

在每个选择的 cnt 个区间内选择一个点，一定有一种选法使得所有区间都包含任意一个点，也就是说只要选择 cnt 个点就能占有所有的区间

假设答案是 ans 且 ans 大于 cnt，如果要选一些点能够占有所有的区间，那么由于选择的 ans 个区间都是两两不相交的，所以说至少需要 ans 个点才可以占有所有的区间，然而 cnt 小于 ans，并不是至少需要 ans 个，原假设不成立，所以 cnt 大于等于 ans 成立

简单来说，由于贪心算法每次选择的是当前最早结束且不与之前选择区间重叠的区间，因此在每一步选择时，贪心选择不会错过任何一个可能的区间来达到 ans 的数量

综上，cnt = ans

### 例题：区间分组

给定 n 个区间，将这些区间分成若干组使得组内部的区间没有交集且组数最少，也就是说有交集的区间一定要分在不同的组内，输出最少组数

![[QQ_1732805551389.png]]

1. 将所有区间按左端点从小到大排序

1. 从前往后枚举区间  
    每次判断是否能放入现有的组（判断该区间左端点是否大于组的右端点）
    
    1. 如果可以则放入（更新组的右端点）
    
    1. 如果不行则新开组
    

按照上述方法分组，假设这样得到的答案是 cnt，正确的最小组数是 ans

**如何证明 cnt 大于等于 ans**

根据贪心选择策略，得到组数一定合法，所以显然大于等于最小组数的合法方案 ans

**如何证明 cnt 大于等于 ans**

一定有一个点同时在 ans 个区间内，所以在分组的时候一定要分出大于 ans 个区间的组才能将 ans 个区间分开，所以选出的组数 cnt 一定大于等于 ans

简单来说，贪心算法在分配时总是尽可能利用现有的组，因此不会使用超过 ans 个组。具体来说，在任何时刻，同时存在的最多重叠区间数决定了最小需要的组数，而贪心算法正好以这种方式分配组数

综上 cnt = ans

# 二、霍夫曼树

### 霍夫曼树

> [!info] 霍夫曼树 - OI Wiki  
> OI Wiki 是一个编程竞赛知识整合站点，提供有趣又实用的编程竞赛知识以及其他有帮助的内容，帮助广大编程竞赛爱好者更快更深入地学习编程竞赛  
> [https://oi-wiki.org/ds/huffman-tree/](https://oi-wiki.org/ds/huffman-tree/)  

如果一个二叉树，每个叶节点的权 $w$ 乘上该叶节点到根节点的距离 $l$ 的总和就是树的带权路径长度，简称= WPL

$$WPL=\sum_{i=1}^nw_il_i$$

给定一堆叶节点的权值，构成 WPL 最小的二叉树就是霍夫曼树，构造霍夫曼树的算法就是霍夫曼算法，基本步骤如下：

1. 初始化：将每一个给定的叶节点都当作一个只有一个根节点的二叉树

1. 选取与合并：从二叉树集合中选根节点权最小的两棵，分别作为左右子树构造一棵新的二叉树，新二叉树的根节点的权值为左右子树根结点的权值和

1. 删除与加入：从二叉树集合中删除作为左右子树的两棵二叉树，并将新建立的二叉树加入集合

1. 重复 2、3 步，当集合中只剩下一棵二叉树时，这棵二叉树就是霍夫曼树

### 例题：合并果子

[https://www.luogu.com.cn/problem/P1090](https://www.luogu.com.cn/problem/P1090)

有 $n$ 堆果子以及果子的重量，每次可以选任意两堆合并，合并的代价是这两堆果子的重量之和，求一种代价最小的将所有果子合并成一堆的路径

经典的霍夫曼树问题，每次找到最小的两个/堆果子进行合并即可，可以用小根堆快速求解

每次弹出两次堆顶，累加到代价，然后在放入合并后的值

![[QQ_1732807580244.png]]

```C++
\#include <bits/stdc++.h>
using namespace std;
const int N = 10005;
int main() {
  int n,x,res=0;
  priority_queue<int,vector<int>,greater<int> > q;
  cin>>n;
  for(int i=1;i<=n;i++)
    q.push((cin>>x,x));
  while(q.size()>1){
    int a=q.top();q.pop();
    int b=q.top();q.pop();
    res+=a+b;
    q.push(a+b);
  }
  cout<<res<<endl;
}
```

# 三、排序不等式

### 例题：排队接水

> [!info] 排队接水 - 洛谷  
>  
> [https://www.luogu.com.cn/problem/P1223](https://www.luogu.com.cn/problem/P1223)  

有 $n$ 个同学排队接水，每个人接水所花时间为 $T_i$，求一个排队顺序使得平均等待时间最小

第 $i$ 个同学会被等 $n-i$ 次，总时间就是：

$$\sum_{i=1}^n T_i(n-i)$$

越靠后权重越小，贪心思想就是：让最长时间的最后打水，代码实现步骤就是排序后从小到大打水

```C++
\#include <bits/stdc++.h>
using namespace std;
struct p{int val,id;} a[1005];
bool cmp(p a,p b){return a.val<b.val;}
int main() {
  long long n,res=0;
  cin>>n;
  for(int i=1;i<=n;i++) cin>>a[i].val,a[i].id=i;
  sort(a+1,a+n+1,cmp);
  for(int i=1;i<=n;i++) res+=a[i].val*(n-i);
  for(int i=1;i<=n;i++) cout<<a[i].id<<' ';
  printf("\n%.2lf\n",(double)res/n);
}
```

**为什么这样做是对的？**

假设已经排序好了 $T_i$ 得到 $T_i'$，假如说此时抽出两项 $T_i'$ 和 $T_j'$ 并且 $T_i'\le T_j'$，即 $i<j$，将这两者交换位置，它们交换前的时间和为 $T_i(n-i)+T_j(n-j)=nT_i-iT_i+nT_j-jT_j$，交换后为 $T_j(n-i)+T_i(n-j)=nT_j-iT_j+nT_i-jT_i$，前式减去后式得 $-iT_i-jT_i+iT_j+jT_i=(j-i)(T_i-T_j)\le 0$，所以任取两项交换后和都会变大，则原式就是最小值

# 四、绝对值不等式

### 例题：货仓选址

> [!info] 货仓选址 - 洛谷  
>  
> [https://www.luogu.com.cn/problem/P10452](https://www.luogu.com.cn/problem/P10452)  

有 $n$ 家商店，每家商店都有一个坐标 $x_i$，现在要给货仓选址 $x$ 使得货仓到所有商店的值和最小

步骤：如果是奇数个数，地址直接取中位数，如果是偶数个，取中位数左右两边任意一个数都可

```C++
\#include <bits/stdc++.h>
using namespace std;
int main() {
  int n,a[100005],loc;
  cin>>n;
  for(int i=1;i<=n;i++) cin>>a[i];
  sort(a+1,a+1+n);
  if(n%2==1) loc=a[(n+1)/2];
  else loc=a[n/2];
  long long res=0;
  for(int i=1;i<=n;i++) res+=abs(a[i]-loc);
  cout<<res<<endl;
}
```

**为什么这样做是对的？**

距离总和的公式为：

$$d=\sum_{i=1}^n|x_i-x|$$

我们可以将第一个点和最后一个点组合，将第二个点和倒数第二个点组合……

$$d=(|x_1-x|+|x_n+x|)+(|x_2-x|+|x_{n-1}-x|)+\dots$$

以第一组为例，如果 $x$ 取在 $x_1$ 和 $x_n$ 之间（可以相等），那么距离只和就是 $x_n-x_1$，往后每一项都是如此，所以说如果偶数个点时 $x$ 取在每一组点的中间就可以使得

$$d=x_n-x_1+x_{n-1}-x_2+\dots$$

如果是奇数个点，取到中位数后离中位数的距离为 $0$，剩下每一组点的距离相当于上述偶数的情况，此时 $d$ 取到最小值

# 五、推公式

### 例题：奶牛玩杂技

> [!info] [USACO05NOV] 奶牛玩杂技 - 洛谷  
>  
> [https://www.luogu.com.cn/problem/P1842](https://www.luogu.com.cn/problem/P1842)  

有一 $n$ 头牛叠罗汉，每头牛都有重量 $W$ 和力量 $S$，那么一头牛的压扁指数就是上方所有牛的重量和减去它的力量，现在要求最大的压扁指数最小

力量大的适合放在下面，重量大的也适合放在下面，所以步骤：按照 $W_i+S_i$ 从小到大排序，就是牛的堆叠顺序（从上到下）

```C++
\#include <bits/stdc++.h>
using namespace std;
struct cow{int w,s,sum,comp;}c[50005];
bool cmp(cow a,cow b){return a.w+a.s<b.w+b.s;}
int main() {
  int n,maxc=-0x3f3f3f3f;
  cin>>n;
  for(int i=1;i<=n;i++)
    cin>>c[i].w>>c[i].s,c[i].comp=-c[i].s;
  sort(c+1,c+1+n,cmp);
  for(int i=1;i<=n;i++){
    for(int j=i-1;j>=1;j--)
      c[i].comp+=c[j].w;
    maxc=max(maxc,c[i].comp);
  }
  cout<<maxc;
}
```

**为什么这样做是对的？**

假设有两头相邻的牛 $i$ 和 $i+1$，且 $W_i+S_i>W_{i+1}+S_{i+1}$，记 $\sum_{k=1}^{i-1}W_k$ 为 $A$ ，那么它们交换前后的压扁指数为：

|   |   |   |
|---|---|---|
||第 $i$ 位置上的牛|$i+1$ 位置的牛|
|交换前|$A-S_i$|$A+W_i-S_{i+1}$|
|交换后|$A-S_{i+1}$|$A+W_{i+1}-S_i$|

将所有数加上 $S_i+S_{i+1}-A$

|   |   |   |
|---|---|---|
||第 $i$ 位置上的牛|$i+1$ 位置的牛|
|交换前|$S_{i+1}$|$W_i+S_i$|
|交换后|$S_i$|$W_{i+1}+S_{i+1}$|

显然 $W_i+S_i>S_i,W_{i+1}+S_{i+1}>S_{i+1}$ 恒成立，又因为有 $W_i+S_i>W_{i+1}+S_{i+1}$，所以 $W_i+S_i$ 同时大于交换后两个位置的压扁质数，所以交换这两头牛后的最大值一定会严格变小

由此可见，只要序列是递增的，就无法进行这样的操作，也就能使得压扁指数最大值最小
