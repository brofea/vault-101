# 三、图的存储

树是无环连通图，树是一种特殊的图。图分为有向图和无向图，无向图是一种特殊的有向图。所以只需要知道如何存储有向图就能知道如何存储所有的树和图

### 边列表

一个结构体存放边，三个成员变量 `u,v,w`，分别存起点、终点、边权

### 邻接矩阵

一个边长为`n`的二维数组`arc`，有权重用`int`无权重用`bool`

![[e3ae3bc75be8f54946f39b0c47e16ce2.png]]

### 邻接表 / 链式前向星

`n`个链表，第`i`个单链表存第`i`个点可以到达的所有点，链中的元素之间的顺序无关紧要，一般插入由`x` 到`y` 的边时，让第`x`个链表的表头指向`y` ，然后让`y`再指向原表的第一个元素，简而言之就是一般在链表的前方插入新边

![[bf6a52670ba5baaed416c725870d48cb.png]]

用多个单链表或向量实现的叫做邻接表，用数组和头节点实现的叫做链式前向星，以下模板的是链式前向星

```C++
// 对于每个点k，开一个单链表，存储k所有可以走到的点。h[k]存储这个单链表的头结点
int h[N], e[N], ne[N], idx;

// 添加一条边a->b
void add(int a, int b)
{
    e[idx] = b, ne[idx] = h[a], h[a] = idx ++ ;
}

// 初始化
idx = 0;
memset(h, -1, sizeof h);
```

### 完全二叉树

完全二叉树可以用顺序存储，用一个数组就可以存储，根节点编号为 1

若节点的下标为 `i` 则：

- 左孩子下标为 `2*i`

- 右孩子下标为 `2*i+1`

这种方法只适合完全二叉树使用，否则会占用大量空间

# 四、图的遍历

### 深度优先遍历

一种特殊的 DFS

```C++
int dfs(int u)
{
    st[u] = true; // st[u] 表示点u已经被遍历过

    for (int i = h[u]; i != -1; i = ne[i])
    {
        int j = e[i];
        if (!st[j]) dfs(j);
    }
}
```

### 拓扑排序

> [OI Wiki - 拓扑排序](https://oi-wiki.org/graph/topo/)  

求拓扑序（Topo）是图的广度优先遍历的经典应用，拓扑序列是一个有向无环图的所有节点排序

如右侧这个图的拓扑序列是 1 2 3。在有向无环图中，所有的边都是由拓扑序列中前方的元素指向后方的元素。有向无环图也被称为拓扑图

在图中，所有元素都有入度和出度，也就是指向的边和被指向的边的数量，如图：

|   |   |   |
|---|---|---|
|节点|入度|出度|
|1|0|2|
|2|1|1|
|3|2|0|

![[QQ_1730546023647.png]]

> **一个有向无环图（DAG, Directed Acyclic Graph）一定存在一个入度为0的点**

在拓扑排序时，任意入度为0的点都可以作为拓扑序的起点，先计算所有点的入度存放在`d[]`中，然后找出所有入度为0的点放入队列`q`中，每次将队头点`x`出队后，迭代找到每个由`x`为起点的边，然后将该边删除，将`x`所指向的那个点的入度减 1，若减 1 后入度为 0，则改点入队

因为有向无环图删除任意一个入度为 0 的点仍然是一个有向无环图，而有向无环图总是有入度为 0 的点，所以上述的迭代操作可以一直进行直到遍历完所有的点

```C++
queue<int> q; // 所有0入度点
vector<int> topo; // 拓扑序答案
while(!q.empty()){
	int tmp=q.front();
	topo.push_back(tmp);
	q.pop();
	while(枚举点tmp所有的出边){
		删除该边
		d[j]--; // 指向的点入度减一
		if(d[j]==0) q.push_back(j);
	}
}
```

# 五、最短路：朴素 Dijikstra

### 最短路问题——引入

> [OI Wiki - 最短路](https://oiwiki.org/graph/shortest-path/)  

给定一个图、起点、终点，求起点到终点的最短距离就是最短路问题

在最短路径算法中，源点代表起点，汇点代表终点。多元源汇就是求大量多个起点和终点的问题

最短路算法可以分为以下类型（ $n$ 表示点数，$m$ 表示边数）

  

**单源最短路 (Single Source Shortest Path, SSSP)**

- 所有边权都为正
    
    - ==朴素 Dijkstra 算法== ==$O(n^2)$==
    
    - ==堆优化 Dijkstra 算法== ==$O(m\log n)$==
    

- 存在负权边（**NSP**）
    
    - ==Bellman-Ford 算法== ==$O(nm)$==
    
    - ==SPFA 算法== ==$O(m)$==
    

**多源汇最短路 (All-Pairs Shortest Path, APSP)**

- ==Floyd 算法== ==$O(n^3)$==

注：朴素 Dijkstra 时间复杂度和边数无关，适合稠密图，$m$ 的数量级大于 $n$ 时使用。堆优化的 Dijkstra 适合稀疏图，$m$ 和 $n$ 在同一个数量级时使用。SPFA算法是 Bellman-Ford 的优化版本，在最坏情况下会达到 $O(nm)$。Dijisktra 基于贪心，另外三者基于 Dp

### 基本思路

> [哔哩哔哩 - 最短路径 迪杰斯特拉 dijkstra算法 数据结构与算法](https://www.bilibili.com/video/BV1QESyYGE55/)

![[QQ_1730733155971.png]]

基本步骤：

- 初始化 dis 数组全为正无限，初始化起点的 dis 为 0

- 循环 n-1 次
    
    - 寻找未确定最短路的点中的 dis 最小的点，记为 t ==$O(n^2)$== ==/== ==$O(n^2)$==
    
    - ==将==点 t 的 final/st 设置为 true ==$O(n)$== ==/== ==$O(n)$==
    
    - 遍历所有以 t 为起点的边，对于每条边，看看边的终点的 dis 是否大于 dis[t] 加该边权，如果是，就更新边的终点的点的 dis ==$O(n^2)$== ==/== ==$O(m)$==
    

- 如果 dis 不是正无限，说明可以从起点出发走到此点，如果是说明无法走到

  

如果需要找到路径，就在每次循环内第三步更新 dis 的时候记录父节点，然后从终点找回到

以上步骤标注的时间复杂度都已经乘上了循环次数 n

使用邻接矩阵的情况下，总的时间复杂度就是 ==$O(n(n+1+n))= O(2n^2+n) \approx O(n^2)$==

使用邻接表的情况下，总的时间复杂度就是 ==$O(n(n+1)+m)= O(n^2+n+m)\approx O(n^2)$==

不难看出朴素 Dijkstra 最慢的一步就是找到未确定最短路 dis 最小的点，这一步就可以用堆/优先队列优化，在下一节会提到

  

### 邻接矩阵模板

```C++
int g[N][N];  // 存储每条边
int dist[N];  // 存储1号点到每个点的最短距离
bool st[N];   // 存储每个点的最短路是否已经确定

// 求1号点到n号点的最短路，如果不存在则返回-1
int dijkstra()
{
    memset(dist, 0x3f, sizeof dist);
    dist[1] = 0;

    for (int i = 0; i < n - 1; i ++ )
    {
        int t = -1;     // 在还未确定最短路的点中，寻找距离最小的点
        for (int j = 1; j <= n; j ++ )
            if (!st[j] && (t == -1 || dist[t] > dist[j]))
                t = j;

        // 用t更新其他点的距离
        for (int j = 1; j <= n; j ++ )
            dist[j] = min(dist[j], dist[t] + g[t][j]);

        st[t] = true;
    }

    if (dist[n] == 0x3f3f3f3f) return -1;
    return dist[n];
}
```

- 在数据小于 $10^9$ 时可以把其初始化为 0x3f3f3f3f，代码可以写成`memset(dist, 0x3f, sizeof dist);`，这样可以防止运算的时候超过数据类型范围

### 邻接表模板

> [落谷 - P3371 【模板】单源最短路径（弱化版）](https://www.luogu.com.cn/problem/P3371)  
>  

```C++
\#include <bits/stdc++.h>
using namespace std;
const int N = 1e4+5; // 最大点数
const int M = 5e5+5; // 最大边数
// 邻接表
int head[N],ver[M]; // 表头 边的终点 
int edge[M],Next[M]; // 边的权值 下一条边的编号
int idx=0;
void add(int u,int v,int w){
  ver[idx]=v;
  edge[idx]=w;
  Next[idx]=head[u];
  head[u]=idx++;
}
// Dijkstra
int dis[N],parent[N]; // 距离数组 父节点
bool vis[N]; // 是否已经是最短路径
int main() {
  // 初始化
  memset(head,-1,sizeof head);
  memset(dis,0x3f,sizeof dis);
  memset(vis,false,sizeof vis);
  // 输入
  int n,m,s; // 点数 边数 起点
  cin>>n>>m>>s;
  for(int i=0;i<m;i++){
    int u,v,w;
    cin>>u>>v>>w;
    add(u,v,w);
  }
  // Dijkstra
  dis[s]=0;
  for(int i=0;i<n-1;i++){ // 循环到n-1即可
    // 1、找到vis为false，距离dis最小的点
    int t=-1;
    for(int j=1;j<=n;j++)
      if(vis[j]==false&&(t==-1 || dis[j]<dis[t]))
        t=j;
    // 2、记录点t已经是最短路
    if(t==-1) break;
    vis[t]=true;
    // 3、用点t更新其他点的距离
    int it=head[t];
    while(it!=-1){
      int nx=ver[it]; // 由点t为起点的边下一个点
      dis[nx]=min(dis[nx],dis[t]+edge[it]);
      it=Next[it];
    }
  }
  // 输出答案
  for(int i=1;i<=n;i++){
    if(dis[i]!=0x3f3f3f3f)
      cout<<dis[i]<<' ';
    else
      cout<<INT_MAX<<' ';
  }
  cout<<endl;
}
```

如果存在重边，就只保留距离最短的，如果存在正自环，则不用管

# 六、最短路：堆优化 Dijkstra

  

### 基本思路

使用堆/优先队列解决循环内的第一步，大大加快了找最小值的速度  

  

- 初始化 dis 数组全为正无限，初始化起点的 dis 为 0

- 循环直到堆空
    
    - 弹出堆顶 dis 最小的点，记为 t，如果已经访问过则略过 ==$O(n)$==
    
    - ==将==点 t 的 final/st 设置为 true ==$O(n)$==
    
    - 遍历所有以 t 为起点的边，对于每条边，看看边的终点的 dis 是否大于 dis[t] 加该边权，如果是，就更新边的终点的点的 dis ==$O(m\log n)$==
    

- 如果 dis 不是正无限，说明可以从起点出发走到此点，如果是说明无法走到

  

以上时间复杂度都乘上了循环次数 n，从小根堆中找最小值的时间复杂度是 $O(1)$ ，每次更新点的 dis 堆需要的时间都是 $O(\log n)$ ，大约每条边都会更新一个点，所以循环内第三步时间复杂度是 $O(m\log n)$ ，总时间复杂度为 ==$O(n+n+m\log n)\approx O(m\log n)$==

### 邻接表+优先队列模板

```C++
typedef pair<int, int> PII;

int n;      // 点的数量
int h[N], w[N], e[N], ne[N], idx;       // 邻接表存储所有边
int dist[N];        // 存储所有点到1号点的距离
bool st[N];     // 存储每个点的最短距离是否已确定

// 求1号点到n号点的最短距离，如果不存在，则返回-1
int dijkstra()
{
    memset(dist, 0x3f, sizeof dist);
    dist[1] = 0;
    priority_queue<PII, vector<PII>, greater<PII>> heap;
    heap.push({0, 1});      // first存储距离，second存储节点编号

    while (heap.size())
    {
        auto t = heap.top();
        heap.pop();

        int ver = t.second, distance = t.first;

        if (st[ver]) continue;
        st[ver] = true;

        for (int i = h[ver]; i != -1; i = ne[i])
        {
            int j = e[i];
            if (dist[j] > distance + w[i])
            {
                dist[j] = distance + w[i];
                heap.push({dist[j], j});
            }
        }
    }

    if (dist[n] == 0x3f3f3f3f) return -1;
    return dist[n];
}
```

注意，无论是在朴素版还是堆优化版的 Dijkstra 中，判断点是否访问过都是在循环内第一步进行的，所以在堆优化版中要在取得堆顶后立即判断点是否已经找到了最短路

在堆优化的外层循环中，使用`while(!q.empty())` 作为循环条件，这个队列其实并不是 dis 数组的代替品，而是一个额外的辅助数据结构，它用于找到最小的 dis，所以同一个点会被加入堆中多次，如果遇到了已经找到最短路的点就直接 `continue`

### 模板题

> [落谷 - P4779 【模板】单源最短路径（标准版）](https://www.luogu.com.cn/problem/P4779)  
>  

```C++
\#include <bits/stdc++.h>
using namespace std;
const int N = 1e5+10;
const int M = 5e5+10;
\#define PII pair<int,int>
\#define fi first
\#define se second
int n; // 点数
// 邻接表
int head[N],ver[M]; // 头节点 边终点
int edge[M],Next[M],idx=0; // 边权 指针
void add(int u,int v,int w){
  ver[idx]=v;
  edge[idx]=w;
  Next[idx]=head[u];
  head[u]=idx++;
}
// dijkstra
void dijstra(int start){
  bool vis[N];
  int dis[N];
  memset(vis,0,sizeof vis);
  memset(dis,0x3f,sizeof dis);
  dis[start]=0;
  // first存储距离，second存储节点编号
  priority_queue<PII,vector<PII>,greater<PII> > q;
  q.push(make_pair(0,start));
  while(!q.empty()){
    // 取出堆顶，即当前距离最小的点
    int tdis=q.top().fi;
    int t=q.top().se;
    q.pop();
    // 判断该点是否已经找到了最短路
    if(vis[t]==true) continue;
    // 将点t标记为已访问
    vis[t]=true;
    // 遍历t的所有边
    int it=head[t];
    while(it!=-1){
      int nx=ver[it];
      int nedge=edge[it];
      if(tdis+nedge<dis[nx]){
        dis[nx]=tdis+nedge;
        q.push(make_pair(dis[nx],nx));
      }
      it=Next[it];
    }
  }
  for(int i=1;i<=n;i++){
    if(dis[i]!=0x3f3f3f3f)
      cout<<dis[i]<<' ';
    else 
      cout<<INT_MAX<<' ';
  }
  cout<<endl;
  return;
}
int main() {
  // 初始化
  ios::sync_with_stdio(false);
  memset(head,-1,sizeof head);
  // 输入
  int m,s; // 边数 起点
  cin>>n>>m>>s;
  for(int i=0;i<m;i++){
    int u,v,w;
    cin>>u>>v>>w;
    add(u,v,w);
  }
  dijstra(s);
}
```

  

# 七、最短路：Bellman-Ford

### 基本思路

Bellman-Ford 算法专注于解决有负权边的 SSSP 问题，有 n 个点和 m 条边，使用边列表储存图，算法步骤如下：

  

- 初始化所有的 dis 为无限大，设置起点 dis 为 0

- 循环 n 次
    
    - 每次遍历所有的边
        
        - 更新最小距离（松弛操作）
        
    

三角不等式：循环 n 次后 dis[b] 一定小于任意 dis[a]+w

然而，NSP 问题棘手的地方在于负权回路，如果负权回路被一直迭代，那么回路上所有点 dis 就会变成负无限，从而导致整个图都更新成负无限

如果第n次迭代仍然会松弛三角不等式，就说明存在一条长度是 n+1 的最短路径，由抽屉原理，路径中至少存在两个相同的点，说明图中存在负权回路

所以不考虑负环的 Bellman-Ford 算法只需要循环 n-1 次，也可以利用第 n 次循环判断是否进行更新了 dis 以判断是否有负环

### 最多经过 k 条边

还可以利用限制循环次数来解 “最多经过 k 条边的最短路” 的题，但是要备份 dis 数组，保证更新时只会使用上一次迭代完所有边的结果

串联的影响如图，在第一次遍历所有边时就更新了 3 号点的距离，所以要先用`memcpy()`备份一份 dis 数组，每次更新使用备份数组判断，修改进 dis 数组。

备份操作需要在循环内第一步进行，也就是遍历所有边之前

`dis[v]=min(dis[v],backup[u]+w);`

![[QQ_1730869853528.png]]

```C++
int n, m;       // n表示点数，m表示边数
int dist[N];        // dist[x]存储1到x的最短路距离

struct Edge     // 边，a表示出点，b表示入点，w表示边的权重
{
    int a, b, w;
}edges[M];

// 求1到n的最短路距离，如果无法从1走到n，则返回-1。
int bellman_ford()
{
    memset(dist, 0x3f, sizeof dist);
    dist[1] = 0;

    // 如果第n次迭代仍然会松弛三角不等式，就说明存在一条长度是n+1的最短路径，由抽屉原理，路径中至少存在两个相同的点，说明图中存在负权回路。
    for (int i = 0; i < n; i ++ )
    {
        for (int j = 0; j < m; j ++ )
        {
            int a = edges[j].a, b = edges[j].b, w = edges[j].w;
            if (dist[b] > dist[a] + w)
                dist[b] = dist[a] + w;
        }
    }

    if (dist[n] > 0x3f3f3f3f / 2) return -1;
    return dist[n];
}
```

# 八、最短路：SPFA

### 基本思路

SPFA 就是 Bellman-Ford 算法的优化。考虑到每次更新最小距离的时候，只有前一个点被更新过了才有可能去更新下一个点，基本思路和 BFS 类似，每次把 dis 更新过（变小过）的节点放入一个队列，然后去更新以队头点为起点的所有边的终点，直到队列为空（没有点可以变小了）

  

- 初始化 dis 数组全为正无限，初始化起点的 dis 为 0

- 向队列中入队起点

- while 队列不为空
    
    - 将队头点 t 取出，对头出队
    
    - 更新 t 的所有出边，如果有点被更新就入队
    

为了防止向队列中加入重复的点，需要用 bool 数组进行记录

```C++
int n;      // 总点数
int h[N], w[N], e[N], ne[N], idx;       // 邻接表存储所有边
int dist[N];        // 存储每个点到1号点的最短距离
bool st[N];     // 存储每个点是否在队列中

// 求1号点到n号点的最短路距离，如果从1号点无法走到n号点则返回-1
int spfa()
{
    memset(dist, 0x3f, sizeof dist);
    dist[1] = 0;

    queue<int> q;
    q.push(1);
    st[1] = true;

    while (q.size())
    {
        auto t = q.front();
        q.pop();

        st[t] = false;

        for (int i = h[t]; i != -1; i = ne[i])
        {
            int j = e[i];
            if (dist[j] > dist[t] + w[i])
            {
                dist[j] = dist[t] + w[i];
                if (!st[j])     // 如果队列中已存在j，则不需要将j重复插入
                {
                    q.push(j);
                    st[j] = true;
                }
            }
        }
    }

    if (dist[n] == 0x3f3f3f3f) return -1;
    return dist[n];
}
```

### 负环问题

SPFA 可以维护一个 cnt[N] 数组，用于记录从起点到该点的最短路径经过的边数，如果在遍历边的时候发现某点的 cnt 大于等于 n 则说明存在负环，因为不存在负环的情况下每个点不可能走两次，cnt 最多是 n-1，此时立即退出外层while 循环。

在用 SPFA找负环的题中，因为没有固定起点，有可能负环和一开始放入队列的一号点并不联通，所以要将所有点都先放入队列中

```C++
int n;      // 总点数
int h[N], w[N], e[N], ne[N], idx;       // 邻接表存储所有边
int dist[N], cnt[N];        // dist[x]存储1号点到x的最短距离，cnt[x]存储1到x的最短路中经过的点数
bool st[N];     // 存储每个点是否在队列中

// 如果存在负环，则返回true，否则返回false。
bool spfa()
{
    // 不需要初始化dist数组
    // 原理：如果某条最短路径上有n个点（除了自己），那么加上自己之后一共有n+1个点，由抽屉原理一定有两个点相同，所以存在环。

    queue<int> q;
    for (int i = 1; i <= n; i ++ )
    {
        q.push(i);
        st[i] = true;
    }

    while (q.size())
    {
        auto t = q.front();
        q.pop();

        st[t] = false;

        for (int i = h[t]; i != -1; i = ne[i])
        {
            int j = e[i];
            if (dist[j] > dist[t] + w[i])
            {
                dist[j] = dist[t] + w[i];
                cnt[j] = cnt[t] + 1;
                // 如果从1号点到x的最短路中包含至少n个点（不包括自己），则说明存在环
                if (cnt[j] >= n) return true;
                if (!st[j])
                {
                    q.push(j);
                    st[j] = true;
                }
            }
        }
    }

    return false;
}
```

Floyd-Warshall 算法的代码实现较为简单，参考下面的代码模板，基本步骤可以抽象理解为：

输入和初始化，若有边 $i\rightarrow j$ 且权重为 $w$，则 $d(i,j)=w$ ，若 $i=j$ 则 $d(i,j)=0$，否则 $d(i,j)=+\infty$

k 代表下图中的红色竖线位置，`d[i][k]`、`d[k][j]`都是红色竖线上的点，然后遍历每一个点，找到这个点在红色竖线上的垂足点，用两个垂足点的和更新距离

![[QQ_1730874049963.png]]

Floyd 算法原理基于动态规划，三维 $d(k,i,j)$ 表示仅能经过 $1～k$ 号点的情况下，由点 $i$ 走到 $j$ 的最短距离

$$d(k,i,j)=min(d(k−1,i,j),d(k−1,i,k)+d(k−1,k,j))$$

优化至二维去掉了 $k$，就是 Floyd 算法

$$d(i,j)=min(d(i,j),d(i,k)+d(k,j))$$

```C++
//初始化：
    for (int i = 1; i <= n; i ++ )
        for (int j = 1; j <= n; j ++ )
            if (i == j) d[i][j] = 0;
            else d[i][j] = INF;

// 算法结束后，d[a][b]表示a到b的最短距离
void floyd()
{
    for (int k = 1; k <= n; k ++ )
        for (int i = 1; i <= n; i ++ )
            for (int j = 1; j <= n; j ++ )
                d[i][j] = min(d[i][j], d[i][k] + d[k][j]);
}
```

# 十、最小生成树：Prim

### 最小生成树问题——引入

给定一个 n 个点的图，求一棵树联通所有点且这颗树的边权和最小，该树将拥有 n-1 条边，而且一定无环，转换成有向边后一定联通。这就是最小生成树问题，在大部分情况下，最小生成树问题都是针对无向图的

> [OI Wiki - 最小生成树](https://oiwiki.org/graph/mst/)  

最小生成树算法分类

- 普利姆算法 Prim 算法，类似 Dijkstra
    
    - 朴素版：稠密图 $O(n^2)$
    
    - 堆优化版：稀疏图 $O(m\log n)$
    

- 克鲁斯卡尔 Kruskal 算法 $O(m\log m)$
    
    - 稀疏图常用，一般不用堆优化 Prim
    

### 基本思路

> [哔哩哔哩 - 图-最小生成树-Prim(普里姆)算法和Kruskal(克鲁斯卡尔)算法](https://www.bilibili.com/video/BV1wG411z79G/)

Prim 的基本思路和 Dijkstra 类似，Dijkstra 是每次找到 dis 最小的没有确定最短距的点，然后遍历其所有出边。Prim 是每次找到离树最近的没有加入到树的的点加入树，也就是找整棵树最小的出边，然后把出边对应的点加入到树中

不过在实际实现中，并不会每次加点都遍历所有的边然后加入点，而是像 Dijkstra 一样维护一个 dis 数组，直接找到 dis 的最小值即可，所以 Prim 算法以找点为核心，时间复杂度和边无关，更适合稠密图

  

- 初始化 dis 为正无限

- 循环 n 次（加点 n 次）
    
    - 找到离集合最近的点 t（ dis最小 ）
    
    - 标记点 t 已经在树中
    
    - 用 t 更新其他点的距离
    

以下代码不像 Djikstra 先选择一个点，所以要循环 n 次而不是 n-1 次。由于要找最小边权和，所以还要维护 res，**而且还要特别注意先累加 res 在去执行第三步的更新，防止有负自环更新了 dis[t] 本身**

**还特别要注意的是，更新距离的时候，更新的不是该点到起点的位置，而是到树的位置**，所以是用边权和 dis 直接做比较：`min(dist[j], g[t][j]);` 而不是像 Dijkstra 一样

```C++
int n;      // n表示点数
int g[N][N];        // 邻接矩阵，存储所有边
int dist[N];        // 存储其他点到当前最小生成树的距离
bool st[N];     // 存储每个点是否已经在生成树中


// 如果图不连通，则返回INF(值是0x3f3f3f3f), 否则返回最小生成树的树边权重之和
int prim()
{
    memset(dist, 0x3f, sizeof dist);

    int res = 0;
    for (int i = 0; i < n; i ++ )
    {
        int t = -1;
        for (int j = 1; j <= n; j ++ )
            if (!st[j] && (t == -1 || dist[t] > dist[j]))
                t = j;

        if (i && dist[t] == INF) return INF;

        if (i) res += dist[t];
        st[t] = true;

        for (int j = 1; j <= n; j ++ ) dist[j] = min(dist[j], g[t][j]);
    }

    return res;
}
```

# 十一、最小生成树：Kruskal

### 基本思路

克鲁斯卡尔算法以加边为核心，每次找到最小的边，判断边的两端在不在一个连通分量里，如果不在就连通，其核心是并查集

  

- 按权排序所有边 $O(m\log m)$

- 枚举每条边 $O(m)$
    
    - 对于边 $a\rightarrow b$，如果点 a b 不连通
        
        - 就将边加入树中
        
    

```C++
int n, m;       // n是点数，m是边数
int p[N];       // 并查集的父节点数组

struct Edge     // 存储边
{
    int a, b, w;

    bool operator< (const Edge &W)const
    {
        return w < W.w;
    }
}edges[M];

int find(int x)     // 并查集核心操作
{
    if (p[x] != x) p[x] = find(p[x]);
    return p[x];
}

int kruskal()
{
    sort(edges, edges + m);

    for (int i = 1; i <= n; i ++ ) p[i] = i;    // 初始化并查集

    int res = 0, cnt = 0;
    for (int i = 0; i < m; i ++ )
    {
        int a = edges[i].a, b = edges[i].b, w = edges[i].w;

        a = find(a), b = find(b);
        if (a != b)     // 如果两个连通块不连通，则将这两个连通块合并
        {
            p[a] = b;
            res += w;
            cnt ++ ;
        }
    }

    if (cnt < n - 1) return INF;
    return res;
}
```

# 十二、二分图：染色法

### 二分图问题——引入

> 二分图，又称二部图，英文名叫 Bipartite graph。
> 
> 二分图是什么？节点由两个集合组成，且两个集合内部没有边的图。
> 
> 换言之，存在一种方案，将节点划分成满足以上性质的两个集合。

二分图的另一个定义就是：不含奇数环的图，奇数环就是包含奇数个点的环

> [OI Wiki - 二分图](https://oiwiki.org/graph/bi-graph/)  

二分图算法分类

- 染色法 $O(n+m)$

- 匈牙利算法 $O(nm)$
    
    - 实际运训时间远小于 $O(nm)$
    

### 基本思路

染色法的作用就是通过判断是否不含奇数环判断一个图是不是二分图

随便找到一个点将其染成白色，然后深搜或广搜找点，每次将未染色的点染成与上一个点相反的颜色，这个过程一定是不会将白色染成黑色或者将黑色染成白色的，因为如果出现染相反色的情况，说明存在奇数环

有 n 个点，伪代码如下：

- flag 初始化为 true

- 循环 n 次
    
    - 如果点 i 未染色
        
        - `dfs(i, 1);`且如果返回了 false，则将 flag 标记为 false，退出循环
        
    

- 如果 flag 为 true，则二分图成功，否则失败

- dfs（将点 u 染色为颜色 c，返回false代表有冲突）
    
    - 将点 u 染色为 c
    
    - 遍历点 u 所有出边，边终点为 j
        
        - 如果点 j 没有染色
            
            - 染色，如果染色冲突则返回 false
            
        
        - 如果点 j 染过色
            
            - 如果颜色和当前点相同则返回 false
            
        
    
    - 返回 true
    

```C++
int n;      // n表示点数
int h[N], e[M], ne[M], idx;     // 邻接表存储图
int color[N];       // 表示每个点的颜色，-1表示未染色，0表示白色，1表示黑色

// 参数：u表示当前节点，c表示当前点的颜色
bool dfs(int u, int c)
{
    color[u] = c;
    for (int i = h[u]; i != -1; i = ne[i])
    {
        int j = e[i];
        if (color[j] == -1)
        {
            if (!dfs(j, !c)) return false;
        }
        else if (color[j] == c) return false;
    }

    return true;
}

bool check()
{
    memset(color, -1, sizeof color);
    bool flag = true;
    for (int i = 1; i <= n; i ++ )
        if (color[i] == -1)
            if (!dfs(i, 0))
            {
                flag = false;
                break;
            }
    return flag;
}
```

# 十三、二分图：匈牙利算法

匈牙利算法可以找到二分图左右两侧最大的匹配数量，也就是说可以找到最大数量的边，使得所有的边都没有共用任何点

### 基本思路

遍历左边的点，尝试匹配右边的点，如果右边的点已经被匹配，就递归尝试将匹配右边的左边点换一个点匹配

> [OI Wiki - 二分图最大权匹配](https://oiwiki.org/graph/graph-matching/bigraph-weight-match/)  

```C++
int n1, n2;     // n1表示第一个集合中的点数，n2表示第二个集合中的点数
int h[N], e[M], ne[M], idx;     // 邻接表存储所有边，匈牙利算法中只会用到从第一个集合指向第二个集合的边，所以这里只用存一个方向的边
int match[N];       // 存储第二个集合中的每个点当前匹配的第一个集合中的点是哪个
bool st[N];     // 表示第二个集合中的每个点是否已经被遍历过

bool find(int x)
{
    for (int i = h[x]; i != -1; i = ne[i])
    {
        int j = e[i];
        if (!st[j])
        {
            st[j] = true;
            if (match[j] == 0 || find(match[j]))
            {
                match[j] = x;
                return true;
            }
        }
    }

    return false;
}

// 求最大匹配数，依次枚举第一个集合中的每个点能否匹配第二个集合中的点
int res = 0;
for (int i = 1; i <= n1; i ++ )
{
    memset(st, false, sizeof st);
    if (find(i)) res ++ ;
}
```
