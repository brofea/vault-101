# 图匹配

## 图匹配问题

在图论中，匹配是指图中一组没有公共端点的边，即在这组边中，任何两条边都不连接到同一个顶点

匹配可以看成“人”和“人之间可能的配对关系”。一条边表示“这两个人可以配对”。匹配就是从这些边里挑出一些，使得任何两个边不能共用端点

在这套体系里，有几种常见的“级别”：

- 极大匹配：已经不能再加边了，但不一定总数最多
- 最大匹配：边数最多的匹配（数量优先）
- 最大权匹配：边有权值，总权值最大（质量优先）
- 最大权最大匹配：在“数量最多”的前提下，让权值和也最大（数量优先再质量）
- 完美匹配：所有点都被匹配上（点数必须是偶数）
- 近完美匹配：只剩一个点没匹配（点数为奇数时的极限情况）

### 完美匹配存在性

这里有两个重要定理：一个管二分图，一个管一般图

### Hall 定理（只对二分图）

假设图的点分成两侧 X 和 Y，只允许连“左边 X 到右边 Y 的边”。我们想要“X 中每个点都匹配到 Y 中某个点”（X-完美匹配）。Hall 定理说

- 对 X 的任意一个点集 W，看它在 Y 里的邻居集合 \(N(W)\)。  
- 必须满足：对所有 W，都有 \(|W| \le |N(W)|\)

意思是：任何一堆左边的点，右边可选的点数都不能比它少，否则不可能“一人一个对象”。若这条条件对所有子集都成立，就一定存在这种完美匹配

### Tutte 定理（一般图）

一般图没左右两边的划分，就用另一种条件：对图任意删去一批点 U，看剩下的图 \(G-U\) 中有多少个“点数为奇数的连通块”，记作 odd\((G-U)\)。Tutte 定理说

- 对所有 U 都要满足：odd\((G-U) \le |U|\)。  
- 这时图才存在完美匹配

它把“完美匹配存在”用一个很抽象但严格的条件刻画出来，你现在只需要知道：Hall 管二分图，Tutte 管通用图

## 二分图最大匹配：匈牙利算法

匈牙利算法可以找到二分图左右两侧最大的匹配数量，也就是说可以找到最大数量的边，使得所有的边都没有共用任何点

### 基本思路

遍历左边的点，尝试匹配右边的点，如果右边的点已经被匹配，就递归尝试将匹配右边的左边点换一个点匹配

> [OI Wiki - 二分图最大权匹配](https://oiwiki.org/graph/graph-matching/bigraph-weight-match/)  

```C++
int n1, n2;     // n1表示第一个集合中的点数，n2表示第二个集合中的点数
int h[N], e[M], ne[M], idx;     // 邻接表存储所有边，匈牙利算法中只会用到从第一个集合指向第二个集合的边，所以这里只用存一个方向的边
int match[N];       // 存储第二个集合中的每个点当前匹配的第一个集合中的点是哪个
bool st[N];     // 表示第二个集合中的每个点是否已经被遍历过

bool find(int x)
{
    for (int i = h[x]; i != -1; i = ne[i])
    {
        int j = e[i];
        if (!st[j])
        {
            st[j] = true;
            if (match[j] == 0 || find(match[j]))
            {
                match[j] = x;
                return true;
            }
        }
    }

    return false;
}

// 求最大匹配数，依次枚举第一个集合中的每个点能否匹配第二个集合中的点
int res = 0;
for (int i = 1; i <= n1; i ++ )
{
    memset(st, false, sizeof st);
    if (find(i)) res ++ ;
}
```
