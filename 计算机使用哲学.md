# 计算机使用哲学

这是关于 brofea 自己使用计算机的规则总结，可能带有强烈的个人偏好和知识局限性

## 文件管理

### 命名规则

1. 所有软件可能调用的文件路径只能使用 `_ - 0-9 a-z A-Z` 共 64 个字符，不能使用空格或中文
2. 使用复数形式，比如 `\samples` 而不是 `\sample`
3. 常见单词使用缩写，比如 `\pic` 而不是 `\pictures`
4. 标识日期直接使用 YYYYMMDD 八个数字，标识月份和年份同理
5. 尽量使用全小写字母

### 用户文件夹

- /dev 代码文件夹

### 资源和归档

对于代码等小体积文件，应该上传 GitHub 管理，这里讨论的是一些大体积文件

- /01-games
  - portable 可以直接运行的便携版游戏
  - packs 游戏安装包
  - assets 游戏资产，按照游戏名分类
  - saves 单独的游戏存档







## 系统使用

- **“下载”的文件随时可以删除**，记得把有用的东西分类放好

- **“桌面”不是存文件的**，桌面只放最近几天会用到的东西，保持桌面文件 / 文件夹少于 5 个

- **使用 App 启动器**，包括 Windows 的 Win 键，macOS 的启动台，不要将应用图标放在桌面

- **开机自启数量应合理**，

### 使用 Windows

- **选择 LTSC 版本**，更稳定，资源占用更低，更新更可控，没有广告，没有多余的预装软件

- **逻辑盘和物理盘一一对应**，一个盘符对应一块硬盘，不给硬盘分区

- **保持默认应用安装路径**，Windows 的应用安装路径已经够多了，不要再安装到别的地方了。当然，如果默认路径太离谱的安装器还是改回 Program Files 吧

- **使用包管理器**，使用 WinGet 和 Scoop 包管理器，能用包管理器安装的软件都用包管理器安装

- **使用便携版或单文件版软件**，如果有的话，尤其是一些体积小巧的小工具

### 使用 macOS

- **使用包管理器**，使用 Homebrew，但能够以 AppBundle 形式存在的软件则不使用包管理器安装

### 使用 Linux

- **绝对不用 Snap 包**。

- **使用 Stow 管理 dotfiles**。

## 硬件使用

- **不要给硬盘分区**，给 SSD 分区没有意义

### 使用笔记本电脑

- **安装厂商的全套驱动程序**，官网里找，而不是用驱动精灵或驱动人生

- **续航很重要**，向 Apple Sillicon 和 Panther Lake 看齐，如果没达到那离电就常开节能模式

### 使用 DIY 主机

- **使用风扇转速管理软件**，如 FanControl (Windows) 和 CoolerControl (Linux)

## 软件使用

- **浏览器用 Chrome**，而不是套壳的 Chromium

- **终端用 Ghostty** 和 Windows Terminal
