# 网络层

本章笔记由 Codex 生成

## 0. 本章学习目标

网络层是计算机网络中内容多、难度高、也很重要的一章。本章重点理解：

- 网络层和网络互连的基本概念。
- IP 地址、子网划分、CIDR 与最长前缀匹配。
- IP 分组交付、路由器转发流程和路由表。
- IP 协议、ARP/RARP、ICMP 的作用。
- Internet 路由选择协议：RIP、OSPF、BGP。
- 路由器的基本结构，以及“转发”和“路由选择”的区别。

## 1. 网络层与网络互连

### 1.1 网络层的主要任务

网络层位于运输层和数据链路层之间，核心任务是把分组从源主机送到目的主机。

主要功能：

- 路由选择：通过路由选择算法，为分组在通信子网中选择合适路径。
- 分组转发：根据路由表或转发表，把分组送往下一跳或目的网络。
- 网络互连：把不同类型的物理网络连接成逻辑上统一的互联网。
- 拥塞控制、差错报告等辅助功能。

### 1.2 网络互连与中间设备

互连网络是利用网桥、路由器等中间设备，把两个或多个物理网络连接起来形成的系统。

常见中继系统：

| 所属层次 | 中间设备 | 作用 |
|---|---|---|
| 物理层 | 转发器、中继器、集线器 | 放大/再生信号 |
| 数据链路层 | 网桥、交换机 | 按 MAC 地址转发帧 |
| 网络层 | 路由器 | 按 IP 地址转发分组 |
| 网络层以上 | 网关 | 协议转换和应用层互连 |

不同网络互连时会遇到很多差异，例如寻址方案、最大分组长度、路由选择技术、差错恢复、用户接入控制、服务类型和管理方式等。IP 协议的意义就在于屏蔽这些异构细节，使各种网络在用户看来像一个统一的虚拟互连网络，也就是 IP 网。

### 1.3 直接交付与间接交付

- 直接交付：目的主机与源主机在同一个网络中，分组可以直接交付给目的主机。
- 间接交付：目的主机在其他网络中，分组需要先交给路由器，再由路由器逐跳转发。

在互联网中，IP 数据报可能经过多个路由器。每个路由器只负责把数据报交给下一跳，而不是一次性知道完整物理传输过程。

## 2. 网络层提供的两种服务

网络层可以向运输层提供两类服务：数据报服务和虚电路服务。

### 2.1 虚电路服务

虚电路服务是面向连接的网络服务。通信前先建立一条逻辑连接，之后同一通信过程中的所有分组沿同一条虚电路传送。

特点：

- 建立连接后，网络像提供一条逻辑“管道”。
- 分组一般按发送顺序到达。
- 每个分组可使用较短的虚电路号，不必携带完整目的地址。
- 若虚电路经过的结点故障，则通过该结点的虚电路会受影响。

注意：虚电路是逻辑连接，不是真正像电话电路交换那样建立物理专线。

### 2.2 数据报服务

数据报服务是无连接的网络服务。网络随时接收主机发送的分组，每个分组独立选择路由。

特点：

- 每个分组都携带目的地址。
- 同一源到同一目的的多个分组可能走不同路径。
- 不保证不丢失、不重复、不乱序，也不保证时限。
- 网络尽最大努力交付，可靠性主要由端系统负责。

Internet 的 IP 层采用的就是数据报服务，即“尽最大努力交付”。

### 2.3 数据报服务与虚电路服务对比

| 对比项 | 虚电路服务 | 数据报服务 |
|---|---|---|
| 基本思路 | 可靠通信由网络保证 | 可靠通信由端系统保证 |
| 连接建立 | 必须建立连接 | 不需要建立连接 |
| 分组地址 | 建立阶段使用完整地址，之后用虚电路号 | 每个分组携带完整目的地址 |
| 转发路径 | 同一虚电路分组走同一路由 | 每个分组独立选择路由 |
| 故障影响 | 故障结点上的虚电路均不可用 | 部分分组丢失，路由可改变 |
| 到达顺序 | 通常按序到达 | 可能乱序 |
| 差错与流量控制 | 可由网络或主机负责 | 主要由主机负责 |

## 3. 网际协议 IP 与 IPv4 地址

### 3.1 IP 协议概述

IP 是 TCP/IP 体系中最核心的协议之一。与 IP 配套使用的协议包括：

- ARP：地址解析协议，把 IP 地址解析为硬件地址。
- RARP：逆地址解析协议，把硬件地址解析为 IP 地址，现已很少使用。
- ICMP：网际控制报文协议，用于差错报告和异常情况报告。
- IGMP：网际组管理协议，用于 IP 多播组管理。

IP 协议特点：

- 不可靠：不保证数据报一定到达。
- 无连接：发送前不建立网络层连接。
- 数据报传送：每个 IP 数据报独立处理。
- 点到点：每一跳是在相邻网络结点之间转发。
- 屏蔽物理网络差异：向上层呈现统一的 IP 网络。

### 3.2 IPv4 地址基本概念

IPv4 地址是 32 位二进制地址，通常写成点分十进制形式，例如：

```text
10000000 00001011 00000011 00011111
= 128.11.3.31
```

基本规则：

- 每个主机或路由器至少有一个 IP 地址。
- Internet 中不能有两个设备使用同一个 IP 地址。
- 如果主机或路由器连接到多个物理网络，可以拥有多个 IP 地址。
- 路由器每个接口通常都有一个属于对应网络的 IP 地址。

两级 IP 地址结构：

```text
IP 地址 ::= { 网络号 net-id, 主机号 host-id }
```

网络号表示主机连接到哪个网络，主机号表示该网络中的哪台主机或路由器接口。

### 3.3 分类的 IP 地址

传统 IPv4 分类地址分为 A、B、C、D、E 五类。

| 类别 | 开头比特 | 地址范围 | 网络号长度 | 主机号长度 | 用途 |
|---|---:|---|---:|---:|---|
| A 类 | `0` | `1.0.0.0` 到 `127.255.255.255` | 7 位 | 24 位 | 大型网络 |
| B 类 | `10` | `128.1.0.0` 到 `191.255.255.255` | 14 位 | 16 位 | 中型网络 |
| C 类 | `110` | `192.0.1.0` 到 `223.255.255.255` | 21 位 | 8 位 | 小型网络 |
| D 类 | `1110` | `224.0.0.0` 到 `239.255.255.255` | 无 | 无 | 多播地址 |
| E 类 | `1111` | `240.0.0.0` 到 `255.255.255.255` | 无 | 无 | 保留/实验 |

常用三类地址容量：

| 类别 | 可用网络数 | 每个网络最大可用主机数 |
|---|---:|---:|
| A 类 | 126 | 16,777,214 |
| B 类 | 16,383 | 65,534 |
| C 类 | 2,097,151 | 254 |

主机号全 0 通常表示网络地址，主机号全 1 通常表示广播地址，因此不能分配给普通主机。

### 3.4 特殊 IP 地址

- 直接广播地址：主机号全 1，用于向某个特定网络上的所有主机广播，只能作为目的地址。
- 受限广播地址：`255.255.255.255`，用于向本网络广播，路由器不转发。
- “这个网络上的特定主机”：网络号全 0，主机号为特定值，只在本网络内部使用。
- 回送地址：网络号为 `127`，用于本机测试和本地进程通信，不应出现在任何网络上。

### 3.5 IP 地址的重要特点

IP 地址是分等级的地址结构，优点是：

- 地址管理方便：地址管理机构只分配网络号，主机号由获得该网络号的单位自行分配。
- 路由表简化：路由器主要根据目的网络号转发，而不是为每台主机维护一条路由。

同一局域网上的主机或路由器接口，IP 地址中的网络号必须相同。路由器连接多个网络，所以通常有两个或多个 IP 地址，每个接口对应一个网络号。

## 4. IP 地址与硬件地址

### 4.1 为什么需要 IP 地址

不同物理网络可能使用不同的硬件地址格式。若直接用硬件地址实现全球互连，就要在各种异构网络之间进行复杂地址转换，几乎不可行。因此互联网在 IP 层使用统一的 IP 地址，由软件屏蔽底层硬件地址差异。

### 4.2 IP 地址和硬件地址的分工

- 网络层及以上：使用 IP 地址。
- 数据链路层及以下：使用硬件地址，也就是 MAC 地址。

IP 数据报在整个源主机到目的主机路径中，源 IP 和目的 IP 基本保持不变；但每经过一个链路，封装它的 MAC 帧源/目的硬件地址都会变化。

例：H1 经过 R1、R2 到 H2。

- IP 层看到的是 `IP1 -> IP2`。
- 第一段链路 MAC 帧可能是 `HA1 -> HA3`。
- 第二段链路 MAC 帧可能是 `HA4 -> HA5`。
- 第三段链路 MAC 帧可能是 `HA6 -> HA2`。

因此，路由器转发时要根据下一跳 IP 地址再找到对应链路上的硬件地址，这正是 ARP 的作用。

## 5. ARP 与 RARP

### 5.1 ARP 的作用

ARP，Address Resolution Protocol，地址解析协议，用于在同一局域网或同一网段内，根据已知 IP 地址找出对应硬件地址。

每台主机都有 ARP 高速缓存，记录本网络中 IP 地址到 MAC 地址的映射。

发送过程：

1. 主机 A 要向目标 IP 发送数据。
2. A 查询自己的 ARP 缓存。
3. 若已有映射，直接把目的 MAC 写入帧首部发送。
4. 若没有映射，A 在本地网段广播 ARP 请求。
5. 目标主机收到后返回 ARP 响应，告诉 A 自己的 MAC 地址。
6. A 更新 ARP 缓存，再发送数据。

### 5.2 ARP 高速缓存的作用

ARP 请求中会带上发送方自己的 IP 和硬件地址。目标主机收到请求后，也可把发送方映射写入自己的 ARP 缓存。这样可以减少之后的 ARP 通信量。

### 5.3 ARP 使用的四种典型情况

| 发送方 | 目的 | ARP 查找对象 |
|---|---|---|
| 主机 | 本网络主机 | 目的主机的 MAC 地址 |
| 主机 | 其他网络主机 | 本网络默认路由器的 MAC 地址 |
| 路由器 | 本网络主机 | 目的主机的 MAC 地址 |
| 路由器 | 其他网络主机 | 本网络下一跳路由器的 MAC 地址 |

要点：ARP 只能解决同一局域网或同一网段上的 IP 到 MAC 映射。跨网络通信时，主机先用 ARP 找到本网络路由器的 MAC 地址，再把分组交给路由器。

### 5.4 RARP

RARP，Reverse Address Resolution Protocol，逆地址解析协议，使只知道自己硬件地址的主机获得 IP 地址。它常用于无盘工作站，但现在已经很少使用。

## 6. IP 数据报格式

IP 数据报由首部和数据两部分组成。

- 固定首部：20 字节，所有 IP 数据报必须有。
- 可变首部：选项字段，长度可变，最多使首部达到 60 字节。

### 6.1 IPv4 首部主要字段

| 字段 | 长度 | 作用 |
|---|---:|---|
| 版本 | 4 位 | IP 协议版本，IPv4 为 4 |
| 首部长度 | 4 位 | 以 4 字节为单位，最大 60 字节 |
| 区分服务 | 8 位 | 用于区分服务，通常不使用 |
| 总长度 | 16 位 | 首部和数据总长度，最大 65535 字节 |
| 标识 | 16 位 | 分片重组时识别同一个原始数据报 |
| 标志 | 3 位 | 其中 MF 表示后面还有分片，DF 表示不允许分片 |
| 片偏移 | 12 位 | 分片在原数据报中的相对位置，以 8 字节为单位 |
| TTL | 8 位 | 数据报最多可经过的路由器数 |
| 协议 | 8 位 | 指明数据部分交给 TCP、UDP、ICMP、IGMP、OSPF 等哪个协议 |
| 首部检验和 | 16 位 | 只检验 IP 首部，不检验数据部分 |
| 源地址 | 32 位 | 源 IP 地址 |
| 目的地址 | 32 位 | 目的 IP 地址 |
| 选项 | 可变 | 支持排错、测量、安全等功能，实际很少使用 |

### 6.2 分片相关字段

当 IP 数据报长度超过链路 MTU 时，路由器可能对其分片。

- 标识字段：让目的主机知道哪些分片属于同一个原数据报。
- MF：`MF = 1` 表示后面还有分片，`MF = 0` 表示最后一个分片。
- DF：`DF = 1` 表示不允许分片，`DF = 0` 才允许分片。
- 片偏移：单位是 8 字节，因此分片数据长度通常要按 8 字节对齐。

课件例子：数据部分 3800 字节，按 1400 字节、1400 字节、1000 字节分成三片，对应片偏移分别为：

```text
第 1 片：0 / 8 = 0
第 2 片：1400 / 8 = 175
第 3 片：2800 / 8 = 350
```

### 6.3 TTL 的作用

TTL，Time To Live，表示数据报在网络中最多可经过的路由器数。每经过一个路由器 TTL 减 1。若 TTL 变为 0，路由器丢弃该数据报，并可用 ICMP 超时报文通知源主机。TTL 可以防止数据报因路由环路在网络中无限循环。

### 6.4 首部检验和

IP 首部检验和只覆盖首部，不覆盖数据部分。发送方把检验和字段置 0，对首部按 16 位反码求和后取反码；接收方重新计算，若结果不为 0 则丢弃数据报。

## 7. IP 层转发分组流程

路由表中最重要的信息通常是：

```text
目的网络地址 -> 下一跳地址 / 输出接口
```

路由器根据目的网络地址确定下一跳，而不是为每台主机维护路由。

### 7.1 普通分组转发算法

1. 从 IP 数据报首部提取目的 IP 地址 `D`，得到目的网络地址 `N`。
2. 若网络 `N` 与路由器直接相连，则直接交付给目的主机。
3. 否则，若有目的地址为 `D` 的特定主机路由，则按该项转发。
4. 否则，若有到达网络 `N` 的路由，则按该项转发给下一跳。
5. 否则，若有默认路由，则交给默认路由器。
6. 否则，报告转发出错。

### 7.2 特定主机路由与默认路由

- 特定主机路由：为某台目的主机指定特殊路径，常用于测试、控制或安全需求。
- 默认路由：当路由表没有匹配项时使用，适合出口较少的网络，可减少路由表大小和查找时间。

## 8. 子网划分

### 8.1 从两级地址到三级地址

早期两级 IP 地址存在问题：

- 地址空间利用率低。
- 每个物理网络都分配网络号会导致路由表太大。
- 两级结构不够灵活。

因此从 1985 年起，引入子网号字段，把两级地址变为三级地址：

```text
IP 地址 ::= { 网络号, 子网号, 主机号 }
```

子网划分是一个单位内部的事情。对外仍表现为一个未划分子网的网络。

### 8.2 子网掩码

IP 数据报首部无法说明目的网络是否划分了子网，因此需要子网掩码。

计算网络地址的基本公式：

```text
(IP 地址) AND (子网掩码) = 网络地址
```

默认子网掩码：

| 地址类别 | 默认子网掩码 |
|---|---|
| A 类 | `255.0.0.0` |
| B 类 | `255.255.0.0` |
| C 类 | `255.255.255.0` |

例 1：

```text
IP 地址：141.14.72.24
子网掩码：255.255.192.0
网络地址：141.14.64.0
```

例 2：

```text
IP 地址：141.14.72.24
子网掩码：255.255.224.0
网络地址：141.14.64.0
```

课件强调：不同子网掩码可能得到相同网络地址，但效果不同。在划分子网时，仅凭 IP 地址不能唯一确定网络地址，必须知道子网掩码。

### 8.3 使用子网掩码的转发过程

在划分子网情况下，路由表项通常包含：

```text
目的网络地址，子网掩码，下一跳
```

转发算法：

1. 从收到的分组中提取目的 IP 地址 `D`。
2. 用直接相连网络的子网掩码与 `D` 逐位 AND，若结果匹配相应网络地址，则直接交付。
3. 若有目的地址为 `D` 的特定主机路由，则按该项转发。
4. 对路由表每一项，用该项子网掩码与 `D` AND；若结果等于该项目的目的网络地址，则按该项转发。
5. 若有默认路由，则按默认路由转发。
6. 否则报告转发出错。

### 8.4 子网规划例题

题目：将 `192.168.3.0` 划分为 5 个子网，每个子网主机数不超过 15 台。

思路：

- 原网络是 C 类，默认前缀 `/24`。
- 至少 5 个子网，需要 3 位子网号，因为 `2^3 = 8`。
- 剩余 5 位主机号，每个子网可用主机数为 `2^5 - 2 = 30`，满足不超过 15 台的要求。
- 子网掩码为 `/27`，即 `255.255.255.224`。

前 5 个子网可规划为：

| 子网网络号 | 可用主机地址范围 |
|---|---|
| `192.168.3.0/27` | `192.168.3.1` 到 `192.168.3.30` |
| `192.168.3.32/27` | `192.168.3.33` 到 `192.168.3.62` |
| `192.168.3.64/27` | `192.168.3.65` 到 `192.168.3.94` |
| `192.168.3.96/27` | `192.168.3.97` 到 `192.168.3.126` |
| `192.168.3.128/27` | `192.168.3.129` 到 `192.168.3.158` |

## 9. CIDR 与构造超网

### 9.1 CIDR 的背景

1992 年前后，Internet 面临三个问题：

- B 类地址即将耗尽。
- 主干网路由表项目急剧增长。
- 整个 IPv4 地址空间最终会耗尽。

在 VLSM 的基础上，提出了 CIDR，Classless Inter-Domain Routing，无分类域间路由选择。

### 9.2 CIDR 的主要特点

CIDR 消除了传统 A、B、C 类地址以及固定子网概念，使用任意长度的网络前缀。

```text
IP 地址 ::= { 网络前缀, 主机号 }
```

CIDR 使用斜线记法：

```text
128.14.32.0/20
```

其中 `/20` 表示网络前缀占 20 位，主机号占 12 位，因此地址块中共有：

```text
2^(32 - 20) = 4096
```

个地址。

`128.14.32.0/20` 的范围：

```text
最小地址：128.14.32.0
最大地址：128.14.47.255
```

### 9.3 路由聚合与超网

一个 CIDR 地址块可以表示很多连续地址。把多个网络聚合成一个路由表项称为路由聚合，也称构成超网。

优点：

- 大幅减少路由表项目数。
- 降低路由器存储和查找开销。
- 减少路由信息交换量。

例：ISP 拥有 `206.0.64.0/18`，可以用一个聚合项表示 64 个 C 类网络；若不用 CIDR，则其他路由器可能需要 64 条路由项。

### 9.4 最长前缀匹配

使用 CIDR 时，路由表项由网络前缀和下一跳组成。查找路由表可能得到多个匹配结果，此时选择网络前缀最长的那个。

最长前缀匹配也叫最长匹配或最佳匹配。

原因：前缀越长，地址块越小，路由越具体。

例：

```text
目的地址 D = 206.0.71.130
路由项 1：206.0.68.0/22
路由项 2：206.0.71.128/25
```

两个都匹配时，选择 `/25`，因为它更具体。

### 9.5 二叉线索查找路由表

当路由表很大时，为提高查找速度，可把无分类编址路由表存放在层次化结构中，常用二叉线索。

基本思想：

- 从 IP 地址最左边的比特开始。
- 比特为 0 走左分支，比特为 1 走右分支。
- 自上而下查找匹配的网络前缀。
- 实际系统中常结合压缩技术提高查找速度。

## 10. ICMP

### 10.1 ICMP 的作用

IP 提供尽最大努力交付，数据报可能丢失、重复、延迟或乱序。ICMP 用于提高 IP 数据报交付成功的机会，让主机或路由器报告差错和异常情况。

要点：

- ICMP 是 IP 层协议，不是高层协议。
- ICMP 报文作为 IP 数据报的数据部分发送。
- ICMP 只能把差错通知出错数据报的源主机。
- ICMP 不负责规定应用程序如何处理这些差错。

ICMP 报文前 4 字节固定包含：

```text
类型，代码，检验和
```

后续 4 字节及数据部分随报文类型变化。

### 10.2 ICMP 报文分类

课件将 ICMP 报文分为：

- 差错报告报文。
- 控制报文。
- 请求与应答报文。

也可简化为两大类：差错报告报文和询问报文。

重要规则：如果携带 ICMP 差错报文的数据报又出错，不再发送新的差错报文，以免引发大量差错报文造成拥塞。

### 10.3 ICMP 差错报文特点

- 通常由路由器发送给源主机。
- 只报告差错，不规定具体处理措施。
- 没有优先权和可靠性保证，也可能丢失。
- 通常伴随着出错 IP 数据报被丢弃而产生。
- 数据区包含出错 IP 数据报的首部和数据区前 8 字节。

不会产生 ICMP 差错报文的情况包括：

- ICMP 差错报文自身出错。
- 目的地址是广播地址或多播地址。
- 链路层广播数据报。
- 不是 IP 分片的第一片。
- 源地址不是单个主机地址，例如零地址、回送地址、广播地址或多播地址。

### 10.4 常见 ICMP 差错报告

目的不可达：

- 网络不可达。
- 主机不可达。
- 协议不可达。
- 端口不可达。
- 需要分片但 DF 置位。
- 源路由失败。

超时：

- TTL 减到 0，路由器丢弃数据报并报告超时。
- 分片重组超时，目的主机丢弃已收到分片。

参数出错：

- IP 首部参数或选项有错误，路由器或目的主机丢弃数据报并报告。

### 10.5 ICMP 控制报文

源站抑制：

- 当路由器处理速度跟不上输入速度，或输出队列拥塞时，可以发送源站抑制报文。
- 源主机收到后降低发送速率。
- 一段时间未收到抑制报文后，可逐步恢复速率。

重定向：

- 当路由器发现源主机有更优下一跳时，向源主机发送重定向报文。
- 使主机维护动态、较小、较优的路径表。
- 只适用于同一网络内网关和主机之间的路径信息交换，不用于网关之间。

### 10.6 ICMP 请求与应答

回送请求与应答：

- 用于测试目的主机或路由器是否可达。
- `ping` 命令就是利用 ICMP 回送请求和回送应答。
- `ping` 是应用层直接使用网络层 ICMP 的例子，不经过 TCP 或 UDP。

时戳请求与应答：

- 用于获得其他主机时钟时间。
- 可估算往返延迟，用于有限的时钟同步。

地址掩码请求与应答：

- 主机启动时可广播请求，获得本网络使用的子网掩码。

### 10.7 ICMP 应用

Ping：

- 发送 ICMP 回送请求。
- 收到回送应答说明目的主机可达，且 IP/ICMP 工作正常。

Traceroute / Tracert：

- 发送 TTL 从 1 开始逐渐增大的数据报。
- 第一个路由器把 TTL 减为 0 后丢弃，并返回 ICMP 超时消息。
- 逐步增大 TTL，即可发现路径上的路由器序列。

路径 MTU 发现：

- 设置 DF，不允许分片。
- 若数据报超过某链路 MTU，路由器丢弃并返回“需要分片但 DF 置位”的 ICMP 报文。
- 源主机逐步减小数据报长度，最终确定路径 MTU。

## 11. Internet 路由选择协议

### 11.1 理想路由算法

理想路由算法应满足：

- 正确、完整。
- 计算简单。
- 能适应通信量和拓扑变化，即具有自适应性。
- 稳定。
- 公平。
- 能选择较优路径。

路由算法可能考虑的参数包括跳数、带宽、时延、负载、可靠性和开销。

### 11.2 静态路由与动态路由

- 静态路由：非自适应，配置简单、开销小，但不能及时适应网络变化。
- 动态路由：自适应，能较好适应网络状态变化，但实现复杂、开销较大。

### 11.3 分层路由与自治系统

Internet 规模太大，不能让所有路由器都知道所有网络的完整路由。因此采用分层次路由选择协议。

自治系统 AS，Autonomous System，是由一个行政单位管理的网络集合。AS 有权自主决定内部使用的路由协议，且 AS 内所有路由器应连通。

两大类路由选择协议：

| 类型 | 使用范围 | 代表协议 |
|---|---|---|
| IGP 内部网关协议 | 一个 AS 内部 | RIP、OSPF |
| EGP 外部网关协议 | AS 之间 | BGP-4 |

## 12. RIP

### 12.1 RIP 基本思想

RIP，Routing Information Protocol，是内部网关协议中最早广泛使用的协议。它是分布式、基于距离向量的路由选择协议。

RIP 中的距离也叫跳数：

- 到直接连接网络的距离为 1。
- 到非直接连接网络的距离为经过的路由器数加 1。
- 最大有效距离为 15。
- 距离 16 表示不可达。

因此 RIP 只适合小型互联网。

### 12.2 RIP 三个要点

- 只和相邻路由器交换信息。
- 交换的信息是自己当前知道的全部路由表。
- 按固定时间间隔交换，例如每 30 秒。

RIP 协议使用 UDP 传送，端口号为 520。

### 12.3 距离向量更新算法

路由器收到相邻路由器 X 的 RIP 报文后：

1. 把收到报文中所有项目的下一跳改为 X，距离加 1。
2. 对每个目的网络：
   - 若本路由表没有该目的网络，则加入。
   - 若已有项目且下一跳也是 X，则用新项目替换旧项目。
   - 若已有项目但下一跳不同，且新距离更小，则更新。
   - 否则不改变。
3. 若 3 分钟没有收到某相邻路由器更新，则把该路由器相关距离置为 16，表示不可达。

### 12.4 RIP 的优缺点

优点：

- 实现简单。
- 开销较小。
- 在小型网络中容易部署。

缺点：

- 以跳数为度量，可能选择非最优路径。
- 最大距离 15，不适合大网络。
- 收敛慢，网络故障传播可能需要较长时间。
- “好消息传播得快，坏消息传播得慢”。
- 安全性较差。
- 早期 RIP 不支持无类地址和 VLSM。
- 周期性发送完整路由表，消耗带宽。

### 12.5 路由循环与处理措施

RIP 可能出现计数到无穷大问题。网络故障后，路由器之间可能互相误以为对方有路径，距离逐步增加到 16 才确认不可达。

常见处理措施：

- 水平分割：从某接口学到的路由，不再从该接口发回。
- 毒性逆转：失效路由不立即删除，而是以 16 的度量值广播。
- 触发更新：路由变化时立即发送更新，而不等待周期。
- 抑制计时：路由失效后一段时间内不接受关于同一目的的可疑更新。

这些措施只能减少路由循环，不能完全消除。

## 13. OSPF

### 13.1 OSPF 基本特点

OSPF，Open Shortest Path First，开放最短路径优先协议，是内部网关协议。它使用 Dijkstra 最短路径算法，是分布式链路状态协议。

“开放”表示协议公开，不受某厂商控制。

### 13.2 OSPF 三个要点

- 使用洪泛法向本 AS 内所有路由器发送信息。
- 发送的信息是本路由器相邻链路状态及其度量。
- 只有链路状态变化时，才用洪泛法发送更新。

所有路由器最终建立一致的链路状态数据库。该数据库相当于全网拓扑图，路由器据此计算最短路径树并生成路由表。

### 13.3 OSPF 区域

为了适应大规模网络，OSPF 把一个自治系统划分为多个区域。每个区域有 32 位区域标识符。

区域划分的好处：

- 把洪泛范围限制在区域内部。
- 减少整个网络通信量。
- 区域内部路由器只需知道本区域完整拓扑。

OSPF 采用层次结构：

- 主干区域：`0.0.0.0`，负责连通其他区域。
- 区域边界路由器：连接多个区域。
- 主干路由器：位于主干区域。

### 13.4 OSPF 与 RIP 的区别

| 对比项 | RIP | OSPF |
|---|---|---|
| 类型 | 距离向量 | 链路状态 |
| 度量 | 跳数 | 链路代价 |
| 更新范围 | 相邻路由器 | 洪泛到区域内/AS 内 |
| 更新内容 | 整张路由表 | 链路状态 |
| 收敛速度 | 较慢 | 较快 |
| 规模适应性 | 小型网络 | 大型网络 |
| 支持 CIDR/VLSM | 早期支持弱 | 支持 |

OSPF 不使用 UDP，而是直接用 IP 数据报传送，因此位置在网络层。

### 13.5 OSPF 分组类型

OSPF 有五类分组：

1. Hello 分组：发现和维护邻居可达性。
2. 数据库描述分组：描述链路状态数据库摘要。
3. 链路状态请求分组：请求某些链路状态信息。
4. 链路状态更新分组：用洪泛法更新链路状态。
5. 链路状态确认分组：确认收到更新。

### 13.6 OSPF 其他特点

- 支持多路径负载平衡。
- 支持鉴别。
- 支持可变长度子网划分和 CIDR。
- 链路状态带有序号，序号越大越新。
- 定期刷新链路状态数据库。
- 响应网络变化快，没有 RIP 中“坏消息传播得慢”的问题。
- 多点接入局域网中可选指定路由器，减少广播信息量。

## 14. BGP

### 14.1 BGP 使用环境

BGP，Border Gateway Protocol，是自治系统之间交换路由信息的外部网关协议。常用版本是 BGP-4。

AS 之间路由选择非常复杂，因为：

- Internet 规模太大。
- AS 之间不一定追求技术意义上的最短路径。
- 必须考虑管理策略、商业关系和安全策略。

因此 BGP 目标不是寻找最佳路由，而是寻找一条能够到达目的网络且较好的、不会兜圈子的路由。

### 14.2 BGP 发言人与会话

每个 AS 至少选择一个路由器作为 BGP 发言人。两个 BGP 发言人通过 TCP 连接建立 BGP 会话，在会话中交换路由信息。

使用 TCP 的好处：

- 提供可靠传输。
- 简化 BGP 自身设计。

BGP 发言人之间互称邻站或对等站。

### 14.3 BGP 交换的信息

BGP 交换的是网络可达性信息，即到达某个网络需要经过的一系列 AS。这种方法称为路径向量。

BGP 路由表应包括：

- 目的网络前缀。
- 下一跳路由器。
- 到达目的网络需要经过的 AS 序列。

BGP 支持 CIDR，有利于路由聚合。

### 14.4 BGP 报文

BGP-4 使用四种主要报文：

| 报文 | 作用 |
|---|---|
| OPEN | 与相邻 BGP 发言人建立关系 |
| UPDATE | 通告新路由或撤销旧路由 |
| KEEPALIVE | 确认 OPEN 并周期性保持邻站关系 |
| NOTIFICATION | 报告检测到的差错 |

RFC 2918 还增加了 ROUTE-REFRESH 报文，用于请求对等端重新通告路由。

## 15. 路由器结构与作用

### 15.1 路由器的任务

路由器是具有多个输入端口和输出端口的专用计算机。其任务是把输入端口收到的分组，根据目的网络，从合适的输出端口转发给下一跳路由器或目的网络。

### 15.2 路由器基本组成

典型路由器包括：

- 输入端口：接收帧，去掉链路层首尾部，把 IP 分组送入网络层处理队列。
- 输出端口：缓存分组，加上链路层首尾部，发送到线路。
- 交换结构：把分组从输入端口送到输出端口。
- 路由选择处理机：运行路由选择协议，维护路由表。
- 转发表：用于高速查表和分组转发。

交换结构常见方式：

- 通过存储器。
- 通过总线。
- 通过互连网络。

### 15.3 转发与路由选择的区别

- 转发：根据转发表，把用户 IP 数据报从合适端口发出去，是逐分组、局部、快速的数据平面动作。
- 路由选择：根据路由选择协议和网络拓扑变化，动态计算路由，是控制平面动作。

路由表由路由选择算法得出，转发表通常从路由表生成。学习原理时有时不严格区分二者，但概念上应分清。

### 15.4 分组排队与丢弃

输入端口或输出端口都可能产生排队时延。当分组进入队列的速度超过路由器处理或输出速度，队列空间耗尽时，后续分组会被丢弃。路由器队列溢出是 IP 分组丢失的重要原因。

## 16. 本章重点速记

- IP 层提供无连接、不可靠、尽最大努力交付的数据报服务。
- 虚拟互连网络通过 IP 协议屏蔽物理网络差异。
- IP 地址用于网络层及以上，MAC 地址用于链路层及以下。
- IP 数据报在端到端路径中源/目的 IP 不变，但每一跳的 MAC 帧地址会变。
- ARP 只解决同一网段内 IP 地址到 MAC 地址的映射。
- IP 首部固定部分 20 字节，最大首部长度 60 字节。
- TTL 防止分组在路由环路中无限循环。
- 子网划分公式：`IP 地址 AND 子网掩码 = 网络地址`。
- CIDR 使用网络前缀和斜线记法，支持路由聚合。
- CIDR 查表采用最长前缀匹配。
- ICMP 是 IP 层协议，用于差错报告、控制和请求/应答。
- `ping` 使用 ICMP 回送请求/应答。
- `traceroute` 利用 TTL 递增和 ICMP 超时报文发现路径。
- IGP 用于 AS 内部，代表协议 RIP、OSPF。
- EGP 用于 AS 之间，代表协议 BGP。
- RIP 基于距离向量，简单但规模小、收敛慢。
- OSPF 基于链路状态，收敛快，适合大规模网络。
- BGP 基于路径向量，目标是按策略找到可达且较好的 AS 间路由。
- 转发是查表把分组送出去，路由选择是计算和维护路径。
