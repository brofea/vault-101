## HTTP

WWW 即 World Wide Web，万维网。核心由下面三个标准构成：

- 统一资源定位符 URL
- 超文本传输协议 HTTP
- 超文本标记语言 HTML

HTTP 即 HyperText Transfer Protocol，超文本传输协议，是 WWW 的核心协议，定义了客户端和服务器之间的通信规则和数据格式

### HTTP 请求报文

包含如下几个部分

- 起始行 (Start Line)
  - 方法
  - URL
  - HTTP 版本
- 请求头 (Headers)，一组键值对
  - Host 目标服务器地址
  - User-Agent 客户端软件信息
  - Accept 客户端可接受的响应内容类型
  - Content-Type 请求体的内容类型
  - Content-Length 请求体的长度
- 一行空行，用于分隔请求头和请求体
- 请求体 (Body)，可选，包含要发送的数据

请求头的字段多种多样，不止上述列出的几个

### HTTP 响应报文

包含如下几个部分

- 起始行 (Start Line)
  - HTTP 版本
  - 状态码
  - 状态消息
- 响应头 (Headers)，一组键值对
  - Content-Type 响应体的内容类型
  - Content-Length 响应体的长度
  - Server 服务器软件信息
- 一行空行，用于分隔响应头和响应体
- 响应体 (Body)，可选，包含要返回的数据

### RESTful API

REST 即 Representational State Transfer，表述性状态转移，是一种设计 API 的风格，强调资源的概念和使用 HTTP 方法来操作资源

一个 RESTful API 的设计应该遵循以下原则：

- 资源导向：每个资源都有一个唯一的 URL
- 无状态：每个请求都应该包含所有必要的信息，服务器不应该存储客户端的状态
- 使用 HTTP 方法：使用 GET、POST、PUT、PATCH、DELETE 等 HTTP 方法来操作资源
- 使用标准状态码：使用 200、201、400、404、500 等 HTTP 状态码来表示请求的结果
- 支持多种数据格式：通常使用 JSON 作为数据交换格式

一个 RESTful 包含以下几个部分：

- 资源：URL 中表示资源的部分，例如 `/users` 表示用户资源，这里不能出现动词
- HTTP 方法：表示对资源的操作，例如 GET 表示获取资源，POST 表示创建资源，PUT 表示替换资源，PATCH 表示部分更新资源，DELETE 表示删除资源
- 状态码：表示请求的结果，例如 200 表示成功，201 表示创建成功，400 表示请求错误，404 表示资源未找到，500 表示服务器错误
- 表述：资源的具体表示形式，例如 JSON 格式的数据
