## 基本语法

### 包的导入导出

在Go中，程序的基本单位是包（package）。每个Go程序都是由一个或多个包组成的。一个包就是一个文件夹，可以包含多个`.go`文件。每个`.go`文件都必须以`package`关键字开头，指定该文件所属的包。例如：

```go
package main
```

使用`import`关键字可以导入其他包中的函数、变量等。例如：

```go
import "fmt"                     // 导入标准库中的fmt包
import "math/rand"               // 导入标准库中的math/rand包
import "github.com/user/package" // 导入第三方包

import (
    "fmt"
    "math/rand"
    "github.com/user/package"
)                                // 一次性导入多个包
```

一个包的导出，也就是函数和变量对其他包的可见性，是通过首字母大小写来控制的。首字母大写的函数或变量是导出的，可以被其他包访问；首字母小写的函数或变量是未导出的，只能在当前包内使用。例如：

```go
func ExportedFunction() {}      // 导出函数
func unexportedFunction() {}    // 未导出函数
var ExportedVariable int        // 导出变量
var unexportedVariable int      // 未导出变量
```

### 运算符

大部分和 C 语言类似，但有如下不同：

- 没有 `++` 和 `--` 运算符，他们只能作为语句使用
- 没有 `?:` 三目运算符
- 增加 `&^` 位清除运算符，`a &^ b` 等同 于 `a & ~b`
- 增加 `<-` 通道操作符
- 增加 `:=` 短变量声明运算符

## 数据类型

Go 是强类型语言，变量必须声明类型，且类型不能改变

### 基本类型

- bool，布尔值，取值为 true 或 false，不能为 0 或 1
- uint8、uint16、uint32、uint64，无符号整数
- int8、int16、int32、int64，带符号整数
- int、uint，平台相关整数类型，32 位系统为 32 位，64 位系统为 64 位
- uintptr，无符号整数类型，用于存储指针，是一个纯数值
- float32、float64，单精度和双精度浮点数（IEEE 754）
- complex64、complex128，复数类型，分别由 float32 和 float64 组成
- byte, uint8 的别名，表示一个字节，rune，int32 的别名，表示一个 Unicode 码点
- string，字符串，可转为字符切片

### 派生类型

- [N]T，数组类型，长度为 N 的 T 类型数组
- []T，切片类型，T 类型的动态数组
- map[T1]T2，映射类型，键为 T1 值为 T2
- type T struct { ... }，结构体类型
- type T interface { ... }，接口类型
- *T，指针类型，指向 T 类型的值
- chan T，通道类型，传递 T 类型的值
- func(T1...) T2，函数类型，参数为 T1 类型，返回值为 T2 类型

### 零值

zero value 是 Go 中每个类型的默认值，当变量声明但未初始化时会被赋予零值。不同类型的零值如下：

- 数值类型（int、float、complex）：0 或 0.0
- 布尔类型（bool）：false
- 字符串类型（string）：""
- 指针、切片、映射、通道、函数、接口类型：nil
- 结构体类型：每个子字段都被赋予对应的零值
- 数组类型：每个元素都被赋予对应的零值

> nil 是一个预声明的标识符，表示一个无效的值，不等同于其他语言的 null 或 None

## 变量和常量

变量声明，使用 var 关键字

```go
var n int
var x, y, z float64 = 1.0, 2.0, 3.0
var (
    a int
    b string
    c bool
)
```

变量赋值

```go
n = 20
x, y, z = 4.0, 5.0, 6.0
a, b, c = 1, "hello", true

```

短变量声明，可省略 var 和类型

```go
x := "string"
u, v := 7, 8

```

变量有如下规则

- 局部变量必须使用，否则编译错误
- 全局变量可以不使用，但会被编译器警告
- 短变量声明的左侧至少有一个新变量，旧变量会被重新赋值

常量声明，使用 const 关键字

```go
const Pi float64 = 3.14  // 常量声明时必须初始化
const e = 2.718          // 可省略类型
const (
    StatusOK = 200
    StatusNotFound = 404
)
```

## CLI输入输出

输出比较常用的有三种方法，第一种是调用os.Stdout

```go
os.Stdout.WriteString("Hello 世界!")
```

第二种是使用内置函数println

```go
println("Hello 世界!")
```

第三种也是最推荐的一种就是调用fmt包下的Println函数

```go
fmt.Println("Hello 世界!")
```

fmt.Println会用到反射，因此输出的内容通常更容易使人阅读，不过性能很差强人意。

此外 fmt.Printf函数可以使用格式化字符串输出，类似于C语言的printf函数

```go
fmt.Printf("Hello %s!", "世界")  // Hello 世界!
fmt.Printf("Pi is approximately %.2f", 3.14159)  // Pi is approximately 3.14
``` 

输入常用的有两种方法，第一种是调用os.Stdin，只能读取字节流，需要自己处理缓冲和转换

```go
var input string
os.Stdin.Read([]byte(input))
```

第二种是调用fmt包下的Scanln函数

```go
// 扫描从os.Stdin读入的文本，根据空格分隔，换行也被当作空格
func Scan(a ...any) (n int, err error) 

// 与Scan类似，但是遇到换行停止扫描
func Scanln(a ...any) (n int, err error)

// 根据格式化的字符串扫描
func Scanf(format string, a ...any) (n int, err error)
```

快速读写可以使用bufio包下的Scanner和Writer

```go
scanner := bufio.NewScanner(os.Stdin)
for scanner.Scan() {
    line := scanner.Text()
    fmt.Println(line)
}

writer := bufio.NewWriter(os.Stdout)
defer writer.Flush()
writer.WriteString("Hello 世界!\n")
```

## 条件语句

Go 中的条件语句有 if、switch 和 select，这里仅学习 if 和 switch，select 会在并发章节介绍

### if 和 if else 语句

和 C 语言的类似，但没有括号

```go
if x > 0 {
    fmt.Println("x is positive")
} else if x < 0 {
    fmt.Println("x is negative")
} else {
    fmt.Println("x is zero")
}
```

此外 if 语句可以在条件前面声明一个局部变量，这个变量的作用域仅限于 if 块内

```go
if x := compute(); x > 0 {
    fmt.Println("x is positive")
}
```

### switch case 语句

和 C 语言类似，但无需 break 语句

```go
switch expr {
    case case1:
        statement1
    case case2:
        statement2
    default:
        default statement
}
```

和 if 一样，switch 也可以在条件前面声明一个局部变量，而且 case 可以是一个表达式

```go
switch x := f(); {        // 等同 x := f(); true
    case x == 0:
        fmt.Println("x is zero")
    case x > 0:
        fmt.Println("x is positive")
    default:
        fmt.Println("x is negative")
}
```

### lable 和 goto 语句

同 C 语言，但不推荐使用

```go
func main() {
    i := 0
    Loop:
    if i < 5 {
        fmt.Println(i)
        i++
        goto Loop
    }
}
// 输出 0 1 2 3 4
```

## 循环语句

### for 语句

和 C 语言类似，没有括号

```go
for init statement; expression; post statement {
    execute statement
}

for i := 0; i < 5; i++ {
    fmt.Println(i)
}
```

Golang 没有 while 和 do-while 循环，但可以通过省略 for 语句的某些部分来实现类似的功能

```go
// while 循环
i := 0
for i < 5 {
    fmt.Println(i)
    i++
}
// do-while 循环
i := 0
for {
    fmt.Println(i)
    i++
    if i >= 5 {
        break
    }
}
```

break 和 continue 语句与 C 语言类似，但 Go 语言还支持带标签的 break 和 continue，可以跳出或继续外层循环

### for range 语句

相当于 python 的 for in 和 C++ 的 for (auto x : container)，可以遍历数组、切片、字符串、映射等

```go
// 遍历数组
arr := [5]int{1, 2, 3, 4, 5}
for i, v := range arr {
    fmt.Printf("index: %d, value: %d\n", i, v)
}
// 遍历切片
slice := []string{"a", "b", "c"}
for i, v := range slice {
    fmt.Printf("index: %d, value: %s\n", i, v)
}
// 遍历映射
m := map[string]int{"a": 1, "b": 2, "c": 3}
for k, v := range m {
    fmt.Printf("key: %s, value: %d\n", k, v)
}
```

## 数组

### 数组声明

数组是固定长度的同类型元素的集合，声明时必须指定长度，长度是类型的一部分

```go
var nums [5]int
nums := [5]int{1, 2, 3}     // 未初始化的元素会被赋予零值
nums := new([5]int)         // 返回指针，等同 &([5]int{})

const N = 5
var nums3 [N * 2]int        // 长度必须是字面量、常量或常量表达式
```

### 数组访问、修改和切割

数组可以通过索引访问和修改，还可以用 `:` 切割，类似 python 的切片，但切割后的结果是一个新的数组，范围是左闭右开

```go
nums := [5]int{1, 2, 3, 4, 5}
fmt.Println(nums[0])            // 1
nums[0] = 10
fmt.Println(nums[0])            // 10
sub := nums[1:4]
fmt.Println(sub)                // [2 3 4]
sub = nums[:3]
fmt.Println(sub)                // [10 2 3]
```

### 多维数组

多维数组是数组的数组，声明时需要指定每一维的长度

```go
// 3 行 4 列的二维数组
matrix := [3][4]int
```

## 切片

切片类似 C++ vector 可变数组，但在函数参数传递时是引用传递。切片的底层实则是一个结构体，包含一个指向数组的指针、切片的长度和切片的容量

### 切片的声明和初始化

```go
slice := []int{1, 2, 3}        // 声明并初始化切片
slice := make([]int, 5)        // 创建长度为 5 的切片，元素被赋予零值
slice := make([]int, 3, 5)     // 创建长度为 3，容量为 5 的切片
slice := new([]int)            // 返回指针，等同 &([]int{})
```

切片的访问、修改和切割与数组类似，这里不再赘述

```go

#### 切片相关内置函数

```go
len(slice)      // 返回切片的长度
cap(slice)      // 返回切片的容量
append(slice, elems...)  // 向切片末尾追加元素，返回新的切片
copy(dest, src) // 将 src 切片的内容复制到 dest 切片，返回复制的元素数量
```

### 切片的插入和删除

```go
// 在尾部插入元素 x
slice = append(slice, x)
// 在头部插入元素 x
slice = append([]int{x}, slice...)
// 在索引 i 处插入元素 x
slice = append(slice[:i], append([]int{x}, slice[i:]...)...)

// 在头部删除元素
slice = slice[1:]
// 在尾部删除元素
slice = slice[:len(slice)-1]
// 删除索引 i 处的元素
slice = append(slice[:i], slice[i+1:]...)
// 删除索引 i 到 j (不包含 j) 之间的元素
slice = append(slice[:i], slice[j:]...)
// 删除所有元素
slice = slice[:0]

```

以上所有操作的时间复杂度和 C++ 的 vector 类似，操作尾部为 O(1)，操作头部或中间为 O(n)，删除所有元素为 O(1) 但会保留底层数组的容量

#### 切片的拷贝

```go
// 直接赋值会让两个切片指向同一个底层数组，修改一个切片会影响另一个切片
slice1 := []int{1, 2, 3}
slice2 := slice1
slice2[0] = 10
fmt.Println(slice1)               // [10 2 3]
fmt.Println(slice2)               // [10 2 3]

// 使用 copy 函数可以创建一个新的切片，底层数组不同，修改一个切片不会影响另一个切片
slice1 := []int{1, 2, 3}
slice2 := make([]int, len(slice1))
fmt.Println(copy(slice2, slice1))  // 3
slice2[0] = 10
fmt.Println(slice1)                // [1 2 3]
fmt.Println(slice2)                // [10 2 3]
```

### 切片的遍历

两种方法，第一种是使用 for 循环和索引，第二种是使用 for range 语句

```go
slice := []int{1, 2, 3, 4, 5}
// 使用 for 循环和索引
for i := 0; i < len(slice); i++ {
    fmt.Printf("index: %d, value: %d\n", i, slice[i])
}
// 使用 for range 语句
for i, v := range slice {
    fmt.Printf("index: %d, value: %d\n", i, v)
}
```

### 多维切片

类似 C++ 的 `vector<vector<T>>`，每一行需要单独初始化长度

```go
// 3 行 4 列的二维切片
matrix := make([][]int, 3)
for i := range matrix {
    matrix[i] = make([]int, 4)
}
```

### 数组和切片的底层区别

数组本质是一个固定长度的内存块，而不是 C 中的一个指针，长度是类型的一部分，赋值是深拷贝所有内容

切片本质是一个结构体，包含一个指向数组的指针、切片的长度和切片的容量

```go
type slice struct {
    array unsafe.Pointer // 1. 指向底层数组的指针
    len   int            // 2. 切片的长度
    cap   int            // 3. 切片的容量
}
```

切片的赋值是浅拷贝指针，修改一个切片会影响另一个切片

当对切片执行 append 且 len 即将超过 cap 时，会执行以下操作：

1. 找一块更大的空内存
2. 把旧数组的内容拷贝过去
3. 让切片的 Pointer 指向新内存，并更新 Len 和 Cap
4. 旧数组如果不再被引用，就会被 GC 回收

所以 append 可能导致切片频繁被移动，性能较差，最好一次性分配足够的容量

## 字符串

字符串字面量有两种，均可用于赋值

```go
s1 := "Hello\n世界!"  // 转义字符串，支持转义字符
s2 := `Hello\n世界!`  // 原生字符串，所有字符原样处理
```

string 本质上是一个只读的字节切片，字符串的长度是字节数而不是字符数，字符串可以通过索引访问单个字节，但不能修改字符串中的字节

Golang 中没有 char 类型，只有 byte 和 rune，本质是 uint8 和 uint32 的别名，分别表示一个字节和一个 Unicode 码点

### 字符串的遍历

字符串使用 for range 语句遍历时会按 Unicode 码点遍历，而不是按字节遍历，返回值是字节开头索引和 rune

```go
var str string = "Hello"
for id, char := range str {
    fmt.Printf("str[%d] = %d\t%c\n", id, str[id], char)
}
var str2 string = "你好世界！"
for id, char := range str2 {
    fmt.Printf("str2[%d] = %d\t%c\n", id, str2[id], char)
}
```

输出：

```test
str[0] = 72     H
str[1] = 101    e
str[2] = 108    l
str[3] = 108    l
str[4] = 111    o
str2[0] = 228   你
str2[3] = 229   好
str2[6] = 228   世
str2[9] = 231   界
str2[12] = 239  ！
```

### 转换为切片

string 可以转换为 byte 切片或 rune 切片，前者按字节转换，后者按 Unicode 码点转换。只有在字符串只包含 ASCII 字符时，byte 切片才能正确表示字符

```go
s := "Hello 世界!"
bytes := []byte(s)
runes := []rune(s)
fmt.Printf("bytes: %v\n", bytes)
fmt.Printf("runes: %v\n", runes)
```

输出

```text
bytes: [72 101 108 108 111 32 228 184 150 231 149 140 33]
runes: [72 101 108 108 111 32 19990 30028 33]
```

### len()

len() 函数返回字符串的字节数，而不是字符数

```go
s := "Hello 世界!"
fmt.Println(len(s))  // 13
```

### 字符串的拷贝

字符串的底层是一个结构体，包含一个指针和一个长度

```go
type stringStruct struct {
    ptr *byte
    len int
}
```

如果直接赋值，执行的是浅拷贝，相当于引用，但由于字符串不可修改，所以大部分情况下不会有副作用

```go
s1 := "Hello World!"
s2 := s1
```

如果需要拷贝一个字符串，有两种方法

```go
var s1, s2 string
s1 = "Hello World!"
s2 = string([]byte(s1))  // 通过转换为 byte 切片再转换回字符串实现深拷贝
s2 = string([]rune(s1))  // 通过转换为 rune 切片再转换回字符串实现深拷贝
s2 = strings.Clone(s1)   // 使用 strings 包的 Clone 函数实现深拷贝，Go 1.20+ 可用
```

### 字符串的连接

使用 `+` 运算符连接字符串会创建一个新的字符串，性能较差，尤其是在循环中连接大量字符串时，建议使用 strings.Builder 或 bytes.Buffer 来构建字符串

```go
var builder strings.Builder
builder.WriteString("Hello")
builder.WriteString(" ")
builder.WriteString("世界!")
result := builder.String()
fmt.Println(result)  // Hello 世界!
```

### 字符串的重新赋值

字符串虽然不可修改，但可以整体重新赋值，重新赋值会创建一个新的字符串，原字符串如果不再被引用，就会被 GC 回收

```go
s := "Hello 世界!"
s1 := "Hello Go!"
s = strings.Clone(s1)
s = s1 + s1
s = "Hello " + "World!"
```

上述代码的最后三行都创建了一个新的字符串，原字符串占用的内存会被 GC 回收

## 映射表

Golang 的映射表 (map) 是无序的键值对集合，类似于 Python 的 dict 或 C++ 的 unordered_map

底层实现上，不同于 C++ umap 的拉链法，Golang map 使用哈希桶实现，每一个桶有 8 个槽位，每个槽位存储一个键值对，如果发生哈希冲突，新的键值对会被存储在溢出桶中，且哈希种子完全随机，每次运行程序时都会改变，防止开发者依赖某种顺序

### 映射表的声明和初始化

```go
var name map[keyType]valueType
name := make(map[keyType]valueType)
```

其中 keyType 必须是可比较的类型，valueType 可以是任意类型

```go
mp1 := map[string]int{
   "a": 0,
   "b": 22,
   "c": 33,
}
var mp2 map[string]int
mp3 := make(map[string]int)
```

### 映射表的访问和修改

和 C++ umap 类似，但访问一个不存在的键会返回 valueType 的零值

```go
mp := map[string]int{
   "a": 0,
   "b": 22,
   "c": 33,
}
fmt.Println(mp["a"])  // 0
mp["a"] = 11
fmt.Println(mp["a"])  // 11
fmt.Println(mp["d"])  // 0，int 的零值
```

C++ umap 访问不存在的键会创建一个新的键值对，Go 中访问不存在的键只会返回零值，所以如果需要判断一个键是否存在，可以使用下面的语法

```go
value, ok := mp["d"]
// ok 为 false，表示键 "d" 不存在，value 为 int 的零值 0
// ok 为 true，表示键 "d" 存在，value 为对应的值

// 也可以直接在 if 语句中判断
if value, ok := mp["d"]; ok {
    fmt.Println("Key exists, value:", value)
} else {
    fmt.Println("Key does not exist")
}
```

可以用 for range 语句遍历映射表，遍历的顺序是随机的

```go
for k, v := range mp {
    fmt.Printf("key: %s, value: %d\n", k, v)
}
```

对 map 使用 `len()` 函数会返回 map 中键值对的数量，而不是 map 的容量

### 映射表的删除

使用 `delete()` 函数删除一个键值对，如果键不存在，delete 不会有任何效果

```go
mp := map[string]int{
   "a": 0,
   "b": 22,
   "c": 33,
}
delete(mp, "b")
delete(mp, "d")  // 删除不存在的键，不会有任何效果
fmt.Println(mp)  // map[a:0 c:33]
```

### 不要使用 NaN 作为键

当 keyType 为 float 时，NaN 不能作为键，因为 NaN 不等于自己，无法判断是否存在；正零和负零被视为相同的键，因为它们在比较时被认为是相等的

```go
mp := make(map[float64]int)
mp[math.NaN()] = 1
mp[math.NaN()] = 2
mp[math.NaN()] = 3
delete(mp, math.NaN())  // 无法删除 NaN 键，因为无法找到它
fmt.Println(mp)
for k, v := range mp {
    fmt.Printf("key: %f, value: %d\n", k, v)
}
```

输出

```text
map[NaN:1 NaN:2 NaN:3]
key: NaN, value: 2
key: NaN, value: 1
key: NaN, value: 3
```

### map 模拟 set

map 键不可重复，Golang 没有提供 set 类型，所以可以使用 map[T]struct{} 来模拟 set，其中 struct{} 是一个空结构体，占用零字节内存

```go
set := make(map[string]struct{})
set["a"] = struct{}{}   // 插入元素
delete(set, "a")        // 删除元素
_, ok := set["a"];      // 判断元素是否存在
println(ok)             // false
```

## 指针

类似 C 的指针，但不支持指针运算，且没有野指针和悬空指针的概念，Go 的垃圾回收机制会自动管理内存，避免了许多常见的内存错误

```go
var p *int
x := 10
p = &x            // 获取 x 的地址并赋值给 p
fmt.Println(*p)   // 通过指针访问 x 的值，输出 10
*p = 20           // 通过指针修改 x 的值
fmt.Println(x)    // 输出 20
```

可以用 new() 或 make() 创建指针

```go
new(int)                 // int指针
new(string)              // string指针
new([]int)               // 整型切片指针
make([]int, 10, 100)     // 长度为10，容量100的整型切片 
make(map[string]int, 10) // 容量为10的映射表
make(chan int, 10)       // 缓冲区大小为10的通道
```

## 结构体

Golang 不是 OOP 语言，但支持面向对象的编程范式，结构体是 Go 中的复合数据类型，可以包含多个字段

### 结构体的声明和创建

声明语法如下，字段名同样遵循首字母大小写的导出规则

```go
type Person struct {
    name string
    age, id int
}
```

Golang 没有构造函数的概念，可以通过字面量或工厂函数创建结构体实例

```go
// 字面量创建
p1 := Person{"Alice", 30, 1}         // 按照字段顺序初始化
p2 := Person{name: "Bob", age: 25}   // 通过字段名初始化，未指定的字段会被赋予零值
p3 := new(Person)                    // 返回指针，等同 &Person{}
var p4 Person                        // 声明一个零值结构体，所有字段被赋予零值

// 工厂函数创建
func NewPerson(name string, age int, id int) *Person {
    return &Person{name: name, age: age, id: id}
}
p5 := NewPerson("Charlie", 35, 3)
```

### 结构体的组合

分为显式组合和匿名组合，以下是显式组合的示例

```go
type Address struct {
    city, country string
}
type Employee struct {
    person Person
    addr Address
    salary int
}
```

匿名组合类似继承，直接继承了另一个结构体的字段和方法，以下是匿名组合的示例

```go
type Employee2 struct {
    Person  // 匿名组合，Employee2 直接拥有 Person 的字段和方法
    Address // 匿名组合，Employee2 直接拥有 Address 的字段和方法
    salary int
}
```

以上两个实例中，访问地址的城市方法不同

```go
emp1 := Employee{
    person: Person{name: "Alice", age: 30, id: 1},
    addr: Address{city: "New York", country: "USA"},
    salary: 50000,
}
fmt.Println(emp1.addr.city)    // New York

emp2 := Employee2{
    Person: Person{name: "Bob", age: 25, id: 2},
    Address: Address{city: "Los Angeles", country: "USA"},
    salary: 60000,
}
fmt.Println(emp2.city)         // Los Angeles，直接访问匿名组合的字段
```

### 结构体指针

结构体指针可以通过 `&` 获取，结构体指针不需要解引用或使用 '->' 运算符访问字段，Go 会自动解引用

```go
var p *Person
p = &Person{name: "Alice", age: 30, id: 1}
fmt.Println(p.name)  // 直接访问字段，Go 会自动解引用
```

### 关联方法

Golang 不是 OOP 语言，不能在结构体内写方法，但可以通过关联方法的方式为结构体定义方法

以下是接收器为值类型的方法示例

```go
type User struct {
    Name string
    Age  int
}

// 这不是普通的函数，这是一个方法，(u User) 是接收器
func (u User) SayHello() {
    fmt.Printf("你好，我是 %s\n", u.Name)
}

func main() {
    u := User{Name: "Gemini", Age: 1}
    u.SayHello()
}
```

接收器也可以是指针类型，使用时会自动解引用，当方法需要修改结构体的字段时，接收器必须是指针类型，否则修改不会生效

```go
type User struct {
    Name string
    Age  int
}
func (u *User) AgeChange(delta int) {
    u.Age += delta
}
func main() {
    u := &User{Name: "Gemini", Age: 1}
    u.AgeChange(5)
    fmt.Printf("我今年 %d 岁\n", u.Age)
}
```

### 函数字段

类似函数指针

```go
type Struct struct {
    f func(int, int) int
}
func add(x, y int) int {
    return x + y
}
func main() {
    s := Struct{f: add}
    result := s.f(2, 3)
    fmt.Println(result)  // 5
}
```

### 结构体标签

结构体标签是一个字符串字面量，紧跟在字段声明后面，用于存储元信息，常用于序列化、数据库映射等场景

语法为

```go
type Struct struct {
    val1 T `type1:"key1" type2:"key2"`
    val2 T `type3:"key3"`
}
```

例如

```go
type Programmer struct {
    Name     string `json:"name"`
    Age      int `yaml:"age"`
    Job      string `toml:"job"`
    Language []string `properties:"language"`
}
```
