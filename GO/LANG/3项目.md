# Golang 项目

## 项目结构

Golang 项目的结构没有强制要求，推荐使用以下结构：

```text
my-go-project/
├── cmd/                # 程序的入口点
│   ├── app1/           # 如果有多个二进制文件，每个对应一个子目录
│   │   └── main.go
│   └── app2/
│       └── main.go
├── internal/           # 核心业务逻辑，禁止被外部项目引用
│   ├── auth/           # 认证模块
│   └── service/        # 业务逻辑服务
├── pkg/                # 可以被外部项目引用的库代码
├── api/                # API 定义文件
├── configs/            # 配置文件模板或默认配置
├── deployments/        # 部署相关
├── scripts/            # 编译、安装、分析等脚本
├── test/               # 额外的外部测试数据和测试脚本
├── go.mod              # 依赖管理
└── go.sum
```

存放主要 `.go` 代码的文件夹如下：

- `cmd/`：包含程序的入口点，即 `main package`
- `internal/`：包含核心业务逻辑，禁止被外部项目引用，
- `pkg/`：包含可以被外部项目引用的库代码，如果这个项目不是一个库项目，那么这个目录可以省略
